## 一、轴定义与边界

**B3 在做什么**：把"训练分布"也变成被学习/被进化的对象——通过自动课程、自我对弈、QD/novelty search、coevolution、可验证 evaluator 闭环——让 agent 在**没有外部固定任务集**的条件下持续提升。这与 VZ 的三层关系：

- **R12**：开放学习只能用"代际超越 / 跨任务可比性"度量进步，不能靠 task-success 曲线；这正是 R12 多家族评估的工程实现。
- **R13**：QD / novelty 是 SSL（结构化压缩）一面，AlphaEvolve / AlphaProof 那一支是 RL（强化）一面，两者交替才是开放学习的引擎。
- **R14**：开放课程产生的"区域规则差异"（XLand / POET niche）就是 regime 身份的训练床——regime 不是 prompt 标签，而是从训练分布的**持久区域**里涌现的运行时控制状态。

**轴边界**：开放学习与 self-modification（C1）的差异——B3 关心"任务/环境分布如何被生成"，C1 关心"agent 内部参数/结构如何被修改"。两者交叉于 AlphaEvolve 与 Darwin-Gödel Machine（已归 C1）；本轴只取 AlphaEvolve（搜索目标在 program/algorithm 空间，非 agent 自身参数）。

### B3-1 AlphaEvolve（默认 Y / 已有 PDF）

- 本地 PDF：[`research/papers/dm/alphaevolve-coding-agent-scientific-engineering-discovery-2506.13131.pdf`](../../papers/dm/alphaevolve-coding-agent-scientific-engineering-discovery-2506.13131.pdf)
- **核心贡献**：把 LLM-based code generation + evolutionary search + 一个或多个 evaluator 闭合成 autonomous pipeline，在 verifiable evaluator 驱动下找到 4×4 复矩阵 **48 次标量乘法**（56 年来首次超越 Strassen）、Google 数据中心调度优化、加速器电路简化等里程碑。

### B3-7 DeepSeek-R1（新增 Y）

- 本地 PDF（待下载）：`papers/B3/deepseek-r1-incentivizing-reasoning-rl-2501.12948.pdf`
- **核心贡献**：**R1-Zero** 完全跳过 SFT，只用 RL（GRPO + rule-based reward on math/code）就让 DeepSeek-V3-Base 涌现 long CoT、reflection、self-verification；R1 加冷启 SFT + 多阶段 RL 进一步对齐。开源权重 + 蒸馏到 1.5B-70B 系列。
- **关键技术机制**：**rule-based verifier**（数学正确性 + 代码 unit test）作为 reward — 完全可验证；GRPO（Group Relative Policy Optimization）省去 critic；"aha moment" 长 CoT 在训练中**涌现**，非教出来的；rejection sampling on R1 outputs → SFT data → 蒸馏 small model。

### B3-8 POET（新增 Y）

- 本地 PDF（待下载）：`papers/B3/poet-paired-open-ended-trailblazer-1901.01753.pdf`
- **核心贡献**：**Paired Open-Ended Trailblazer**：env-agent 共演化的奠基作。环境与 agent 同时被进化算法维护；env 通过 **Minimal Criterion Coevolution**（MCC）保留"正好够难"的 niche；transfer step 让一个 niche 的 agent 能跨 niche 适应。
- **关键技术机制**：env 演化 = mutate env params + minimal criterion 过滤（必须有 agent 能解但又不太容易）；agent 演化 = ES on agent params；**transfer step**：尝试把每个 niche agent 跨 niche 部署，若胜出则取代；niche 集合 ≈ MAP-Elites style behavior diversity archive。
- **评分**：理论新颖度 5 / 工程深度 3 / 影响力 4 / VZ 关联度 4。
- **R-ID 命中**：R13（开放学习的 SSL 侧奠基 — 加强）、R14（niche = persistent regime 的环境侧对应 — 补全）、R12（transfer success 作为度量 — 补全）。
- **VZ 行动建议**：**远观背景** + **概念借鉴**。把 "niche" 作为 R14 regime 的环境侧对照——regime 不仅是内部状态分簇，也对应外部环境/对话场景的持久区域。
- **重叠状态**：probe 独家。

### B3-9 PAIRED（新增 Y）

- 本地 PDF（待下载）：`papers/B3/paired-emergent-complexity-uded-2012.02096.pdf`
- **核心贡献**：提出 **Unsupervised Environment Design (UED)** 框架：3-player game 中 adversary 生成 env，protagonist + antagonist 在 env 上跑出 regret，adversary 优化 regret 而非纯难度——保证生成的 env "对当前 agent 难但可解"。形式化证明 regret-optimal env 课程是收敛的。
- **关键技术机制**：3-player setup（env adversary / protagonist / antagonist）；regret = max(antagonist_return) − protagonist_return；adversary 用 RL 优化 regret；理论保证：minimax regret 下 protagonist 收敛到 minimax-optimal policy。
- **评分**：理论新颖度 5 / 工程深度 4 / 影响力 5 / VZ 关联度 5。
- **R-ID 命中**：R12（regret-based curriculum 给"无 ground-truth 任务"的 evaluation 形式工具 — **可直接落地**）、R13（adversary-protagonist 是 SSL ↔ RL 交替的另一种实现 — 加强）、R14（regret signal 可作为 regime 切换条件 — 补全）。
- **VZ 行动建议**：**可立刻反哺**。把 regret-as-curriculum-signal 作为 [`docs/specs/evaluation.md`](../../../docs/specs/evaluation.md) 与 scenario package 生成（参见 [`.cursor/rules/scenario-package-generation.mdc`](../../../.cursor/rules/scenario-package-generation.mdc)）的形式化方法。
- **重叠状态**：probe 独家。
