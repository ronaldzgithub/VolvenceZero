> - **BAYES 组**走"不确定性进入控制"轴 — 把 posterior / confidence 从 readout 升级为经过校准的 plasticity evidence；目标是让系统有原则地决定**吸收多少新知、保留多少先验**，但不改写原始 PE / credit，也不触碰冻结基底。

# 第一部分：DM 组（能力扩展轴，7 条）

## DM-1. PE 输出从 scalar 升级为 distribution

### 现状盲点

当前 [`packages/vz-cognition/`](../../packages/vz-cognition/) 的 prediction-error owner 输出是否包含**价值分布**信息（如 quantile / categorical distribution / 分布矩），还是只输出标量 surprise？需要核查。如果只是 scalar，那么我们错失了**"分布塌缩 = 病理状态"**这一根本健康信号——R7 self-track 的"心理健康"将只能依赖行为代理指标（如对话频率、响应长度），而非有原则的内禀信号。

### 可落地动作

1. 在 [`docs/specs/prediction-error-loop.md`](../specs/prediction-error-loop.md) 加入"PE distributional readout"小节：定义最简实现——PE owner 的 snapshot 输出契约从 `(mean_surprise, ...)` 扩展为 `(mean, distribution_summary)`，其中 `distribution_summary` 至少包含 quantile（如 IQR）+ entropy + asymmetry（skew）。
2. 在 [`docs/specs/lifeform-vitals.md`](../specs/lifeform-vitals.md) 把"价值分布方差/熵/asymmetry 漂移"列为新增 vitals slot，作为 lifeform health 的一级指标。
3. 评估证据先行：在 evaluation 加一族"分布编码完整性"指标（不进入控制主链），先用 SHADOW 模式跑一段时间观察是否能稳定区分"健康对话 vs 关系破裂前兆"。

### 涉及 spec

- [`docs/specs/prediction-error-loop.md`](../specs/prediction-error-loop.md)
- [`docs/specs/lifeform-vitals.md`](../specs/lifeform-vitals.md)
- [`docs/specs/evaluation.md`](../specs/evaluation.md)（新增 readout 族）

# 第二部分：OA 组（安全治理轴，10 条）

> OA 组的共同主题：**当系统开始"会学"以后，怎么保证它不会偷偷学坏**。

## OA-6（P1/M）— 给 metacontroller 加"控制器抗操控"测试（CoT-Control 等价物）

## OA-7（P1/M）— 给 derived 层加 chunky-routing 自检（SURF 思想）

## OA-9（P2/L）— 探索 PE faithfulness 指标（N6 思想 → R-PE 工程化）

### 现状盲点

VZ 的 R-PE 路线宣称"PE 是一级原始信号，控制器响应 PE 而非 token"。但**这个忠实性从未被量化**。N6 把同一个问题在 LLM 的 CoT 上做出了基线实证：CoT 大多数情况下不忠实反映 hint 的使用（reveal rate 多在 1-20%）。如果我们不引入 PE faithfulness 测量指标，永远无法回答"我们的 metacontroller 真的在响应 PE 吗，还是 PE 只是装饰"这个根本问题。

### 可落地动作

1. 给同一 input 构造两个版本：低 PE upstream 和高 PE upstream（通过控制 user_model / belief_assumption 等 owner 状态实现）。
2. 验证 metacontroller 的 z_t / β_t **明显反映**这种 PE 差异（不只是均值不同，整个分布响应应可统计区分）。
3. 度量 PE faithfulness = "高 PE 时控制器响应改变" 的统计频率。
4. 验收阈值：metacontroller 的 PE faithfulness > 80%（远高于 N6 中 LLM CoT faithfulness 多在 1-20%，因为 z_t 在控制器空间，不应被 token 空间表面分布操控）。
5. 这个指标本身**就是 VZ 路线"内部诚信度"的可测信号**——可以反过来当 R-PE 健康度的 dashboard。

# 第三部分：CMA 组（连续记忆证明轴，3 条）

## CMA-1（P2/M-L）— derived layer 引入只读 spreading activation shadow path

### 现状盲点

[`docs/specs/continuum-memory.md`](../specs/continuum-memory.md) 已经把 memory owner 内部重分层成 `learned core + artifact store + derived index`，并且 retrieval 已升级为 owner-side learned-core-guided recall。但当前证明面更偏向"统一 recall signal + artifact anchor"，对**显式多跳联想召回**的机制描述还不够硬。CMA 的 associative routing 探针显示，spreading activation 能在 query 没有词法重叠时，通过 semantic / temporal / structural edges 找到多跳相关记忆；这正好对应我们 derived layer 未来要承担的"关系图谱 / 聚合索引 / 跨 owner readout"能力。

但这条不能直接做成主路径。CMA 的参考实现允许 retrieval 驱动 graph mutation，如果照搬到 VZ，会带来两个风险：(1) 自由 edge mutation 绕过 `MemoryModule` 唯一 owner；(2) 不受控的 activation feedback loop 导致 memory drift。正确姿势是先做**只读 shadow path**，只用于候选重排和 evidence，不写 durable memory。

### 可落地动作

1. 在 [`docs/specs/continuum-memory.md`](../specs/continuum-memory.md) 增加 "Read-only Spreading Activation Shadow Path" 小节，明确该路径只属于 derived layer 的 SHADOW 评估，不参与 durable write。
2. 定义最小 edge 类型：semantic、temporal（FOLLOWED_BY）、associative、structural（owner-to-owner readout），每条 edge 必须带 provenance。
3. 实现 damped propagation：activation 只在 bounded hop 内传播；每次 query 输出 activation trace，作为可审计 telemetry。
4. 只允许把 activation 结果用于 retrieval candidate re-ranking；任何新 edge / edge weight 更新都必须经 session-post slow loop + ModificationGate，不能在 turn-time retrieve 中原地写回。
5. 与 CMA-2 的 VZ-MemProbe 绑定：只有 shadow path 在 associative recall / disambiguation probe 上有稳定收益，才考虑进入 ACTIVE。

- [`docs/specs/continuum-memory.md`](../specs/continuum-memory.md)
- [`docs/specs/evaluation.md`](../specs/evaluation.md)（记录 shadow-path evidence）
- 后续实现位置待 SPLIT 决定，应保持在 memory owner 内部或 derived layer 内部，不新增第二 memory owner

### 预期收益

- 给 R5/R6 的"联想记忆"增加可解释、多跳、可审计的工程路径，而不是只依赖向量相似度。

## CMA-2（P1/M）— 实现 VZ-MemProbe 评估套件（CMA 4 探针 VZ 化）

### 现状盲点

VZ 的 memory spec 很强，`continuum_profile`、multi-band CMS、session-post slow loop、bounded reflection apply、owner-side recall signal 都已经有清晰契约。但我们还缺一个专门面向 R5/R6 的**长时间窗行为级评估套件**：系统是否真的优先新事实？是否能回答"那时还发生了什么"？是否能跨 owner 做多跳联想？是否能在同词不同语境下避免污染？如果没有这组 probe，R5/R6 仍然更多是 spec 信念，而不是可重复证据。

CMA 的 4 个探针（Knowledge Updates / Temporal Association / Associative Recall / Disambiguation）比它的具体实现更值得先吸收，因为它们是 read-only evaluation，不改变 runtime，不触碰 owner 边界，收益高、风险低。

### 可落地动作

把 CMA 的 4 个 probe 改造成 VZ 版本：

1. **VZ-Probe-Update**：用户先给 commitment / preference / belief A，后更新为 B；查询时必须优先返回 B，并能解释 A 已过期。
2. **VZ-Probe-Temporal**：围绕一次 session event 构造前后相邻事件；查询"X 周围还发生什么"时，必须返回时间邻域而不是只返回语义相似项。
3. **VZ-Probe-Assoc**：构造跨 owner 多跳链路（如 user_model → relationship_state → boundary_consent），查询时不能只停留在第一跳。
4. **VZ-Probe-Context**：同一关键词在不同 regime / scene 下有不同意义（如 Python=动物/语言、repair=关系修复/代码修复），检索不能跨语境污染。

## CMA-3（P1-P2/S-M）— ModificationGate 加 memory drift telemetry / anomaly report

### 现状盲点

[`docs/specs/continuum-memory.md`](../specs/continuum-memory.md) 已经要求 memory owner 支持 checkpoint / rollback，并且 reflection durable apply 是 bounded 的。但只要系统允许 promotion、decay、retrieval-induced reinforcement、belief writeback、derived index 重排，就一定存在**memory drift** 风险：错误事实被反复强化、过期记忆持续 dominate retrieval、某个 owner snapshot 在 session-post consolidation 后反复震荡。CMA 明确把 Memory Drift 列为失败模式，并建议记录 provenance、reinforcement history、anomaly scores；这与 VZ R10 ModificationGate 天然对齐。

但 CMA 的建议偏日志层，太浅。VZ 不应只统计"某个 fragment 被 reinforce 次数过多"，而要把 drift 定义在 **PE 没下降 + retrieval dominance 上升 + owner snapshot stability 变差** 的交叉处。

### 可落地动作

1. 在 [`docs/specs/credit-and-self-modification.md`](../specs/credit-and-self-modification.md) 增加 "Memory Drift Telemetry" 小节，作为 ModificationGate 的 read-only 输入，不直接作为 reward。
2. 定义最小 drift signals：
   - `reinforcement_without_pe_drop`：某记忆被持续强化，但相关 PE 没有下降；
   - `retrieval_dominance_spike`：少数 fragments 在多场景中过度 dominate recall；
   - `outdated_fact_reactivation`：已被更新/废弃的事实持续被激活；
   - `owner_snapshot_oscillation`：belief / commitment / relationship_state 等 owner 快照在 slow loop 后反复震荡；
   - `rollback_or_shadow_abort_rate`：memory 相关改动触发 rollback / SHADOW abort 的频率升高。
3. 第一阶段只做 anomaly report，不自动关门；第二阶段再决定是否接入 ModificationGate 阈值。
4. 若推进 CMA-1，则 CMA-3 必须先做或并行做；否则 spreading activation 的反馈回路没有 drift 护栏。

### 涉及 spec

- [`docs/specs/credit-and-self-modification.md`](../specs/credit-and-self-modification.md)
- [`docs/specs/continuum-memory.md`](../specs/continuum-memory.md)
- [`docs/specs/prediction-error-loop.md`](../specs/prediction-error-loop.md)（引用 PE drop / no-drop 的定义）

- 把 memory drift 从"上线后用户感觉不对"提前到 telemetry 层发现。
- 给 CMA-1 / DM-6 / reflection writeback 这类会改变 memory dynamics 的改动加安全垫。
- 与 OA-4 VZ-Audit Agent、OA-3 FramingAwarenessCheck 共同构成 R10 的三层防线：语义 framing、防御审计、记忆漂移。

**D1. Logan J. *Continuum Memory Architectures for Long-Horizon LLM Agents*. arXiv:2601.09913, 2026.**

- 文档位置：[`research/openai-frontier-2026/papers/D1_continuum_memory_architectures.pdf`](../../research/openai-frontier-2026/papers/D1_continuum_memory_architectures.pdf)

## EVO-1（P1/M）— ModificationGate 候选 archive = MAP-Elites + island

### 现状盲点

若 rare-heavy 候选池只做 **标量 net-benefit 排序**，等价演化搜索里的 **单峰爬山** —— 易 **早熟收敛**到单一改动类型，错失「跨家族评估轮廓不同」的优质候选。AlphaEvolve §2.5 强调档案须 **平衡 exploitation 与 exploration**，并用 **MAP-Elites 风格 + island** 维持多样性。

### 可落地动作

1. 在 gate **所属 owner 内部**维护 **行为描述符**（例如 artifact 类型 × 影响面 × rollback 成本 bin × 六大家族评估 digest）→ **elite 集合** 的档案结构。
2. 引入 **island**（可按 rare-heavy / session-medium 划分）与 **岛间限速迁移**，防止单一岛屿支配。
3. 档案与采样策略仅通过 **snapshot** 暴露只读事实；消费者不直接遍历内部可变结构。

# 第六部分：SYS 组（底层机制与架构补强轴，6 条）

> **SYS 组的共同主题**：补充前五组在 Cognitive AGI 和 VZ 核心不变量（R1-R15）中遗漏的底层硬核算法与范式。重点是将高层概念（如涌现、门控、控制、记忆结构）落实为具体的、有论文背书的机制级实现。

## SYS-1（P0/S）— 基于 CPD (Change Point Detection) 的 $\beta_t$ 涌现切换

### 现状盲点

DM-3 提出了用 Interest Function 来触发 Regime 切换，但这仍然需要端到端学习一个函数。真正的"涌现"应该基于底层信号的突变。如果缺乏基于底层信号的无监督切换，$\beta_t$ 的边界依然是启发式或强监督的。

### 可落地动作

1. 在 [`docs/specs/emergent-action-abstraction.md`](../specs/emergent-action-abstraction.md) 中引入 CPD（Change Point Detection）机制作为 $\beta_t$ 切换的核心算法。
2. **直接利用 PE spike（预测误差尖峰）+ Reward Shift** 来无监督地自动检测 Option 的边界。当系统突然预测不准了（PE 飙升），就说明当前的 $\beta_t$ 结束了，触发控制器切换。
3. 这完美契合 R-PE（PE 是一级信号）的哲学，将 PE 信号直接用于时间抽象的边界划分。

### 涉及 spec

- [`docs/specs/emergent-action-abstraction.md`](../specs/emergent-action-abstraction.md)
- [`docs/specs/temporal-abstraction.md`](../specs/temporal-abstraction.md)

### 预期收益

- 让 $\beta_t$ 的切换真正实现"无监督涌现"，摆脱对硬编码规则或强监督 Interest Function 的过度依赖。
- 将 PE 信号的价值最大化，使其不仅用于学习，还直接用于控制流的调度。

### 引用论文

**A4-04. Change Point Detection + Option-Critic. arXiv:2510.24988, 2025.**
- 关键观点：利用 PE spike 和 reward shift 自动检测 option 边界（无需外部监督），这是 $\beta_t$ 切换信号最自然的实现路径。

## SYS-2（P0/M）— ModificationGate 引入 VC 容量上限 (Capacity Bound) 双门机制

### 现状盲点

DM-4 强调了 ModificationGate 必须有"可验证的 Evaluation"（Validation Margin）。但这只防了"改错了"，没防"改得过拟合"。如果一次 rare-heavy 更新改变了太多的模型参数或逻辑容量，系统就会丧失泛化保证。

### 可落地动作

1. 在 [`docs/specs/credit-and-self-modification.md`](../specs/credit-and-self-modification.md) 的 ModificationGate 中引入 **Capacity Cap（修改容量上限）** 概念。
2. 将 ModificationGate 升级为"双门"机制：第一道门看分数（Validation Margin，即 DM-4 的可验证提升），第二道门看容量（Capacity Cap，即本次修改的 VC 维或参数变动量是否超标）。
3. 结合 EWC（Elastic Weight Consolidation）等算法，量化并限制每次自修改对系统整体容量的占用。

### 涉及 spec

- [`docs/specs/credit-and-self-modification.md`](../specs/credit-and-self-modification.md)

### 预期收益

- 在 PAC 学习框架下为系统的长期自修改提供严格的泛化保证。
- 防止系统在持续学习中因为单次过度拟合而导致灾难性遗忘或能力塌陷。

### 引用论文

**C1-01. Two-Gate Guardrail for Self-Modifying Agents. arXiv:2510.04399, 2025.**
- 关键观点：形式化证明自修改 agent 要保留 PAC 学习保证，需要 policy-reachable 模型族 VC 维有界。提出"validation margin + capacity cap"双门机制。

# 第七部分：BAYES 组（校准不确定性驱动可塑性轴，1 条）

> **BAYES 组的共同主题**：吸收 Kalman / recursive Bayesian inference 的核心思想——让不确定性参与"新证据吸收多少"的控制；不照搬完整 Kalman Bayesian Transformer，不把所有 owner 压进一个全局 latent distribution，也不把冻结 substrate 改成在线 Bayesian 权重模型。

## BAYES-1（P1/M）— Precision-aware plasticity：校准不确定性驱动先验/新知平衡

### 现状盲点

当前系统已经有多个名为 uncertainty / confidence / posterior 的量，但它们还不能直接当作 Kalman precision：

- `vz-temporal` 的 `posterior_std` 描述的是 `z_t` 潜在控制代码的不确定性，主要用于采样与 KL；它不是 controller 参数的 epistemic covariance，也不是 observation noise。
- `LearnedUpdateRule.confidence` 是有界 learned gate score，不是经过 coverage / likelihood 校准的概率方差。
- `PredictionErrorSnapshot.pe_decomposition` 已区分 aleatoric / epistemic，但当前按 spec 仍以 report-only 为主；EMA variance 与 learned critic residual 是可用候选，不等于已经识别出的 measurement covariance。
- CMS 已有 ATLAS replay、Titans PE gate、slow→fast backflow 与 checkpoint/rollback，但更新幅度仍主要由 learned scalar gate 决定，没有显式区分"旧状态本身有多不确定"与"这条新证据有多嘈杂"。

因此，真正缺口不是"代码里没有 Kalman 公式"，而是缺少一条**校准 uncertainty → bounded plasticity** 的合法证据链。若在 `P` / `R` / `Q` 没有清晰语义前直接套公式，会把启发式 proxy 包装成伪精确概率。

### 数学边界

仅在 owner-local、低维、观测模型明确的场景采用标量/对角或小型 full-covariance 更新：

```text
P_prior,t = P_posterior,t-1 + Q_change,t
K_t = P_prior,t H_t^T (H_t P_prior,t H_t^T + R_obs,t)^-1
mu_t = mu_prior,t + K_t (y_t - H_t mu_prior,t)
P_posterior,t = (I - K_t H_t) P_prior,t
```

- `P`：owner 对当前 belief / 小型 predictive-head 参数的 epistemic uncertainty。
- `R`：typed outcome / observation 的 aleatoric noise；不能由原始文本或单一 confidence 冒充。
- `Q`：非平稳性 / change-point / regime transition 带来的 process uncertainty。没有 `Q`，递归 covariance 会不断收缩，最终导致 lifeform 过度确信并停止学习。
- `K`：只决定候选更新幅度，不改写原始 PE、outcome 或 credit ledger。

简化式 `K = sigma_prior^2 / (sigma_prior^2 + sigma_obs^2)` 只适用于标量且 `H = 1` 的情形，不能作为跨 owner 通用公式。

### 可落地动作

#### Phase 0：先固定 uncertainty 语义与校准证据

1. 在 [`prediction-error-loop.md`](../specs/prediction-error-loop.md) 中区分：
   - latent/action uncertainty；
   - predictive-head parameter epistemic uncertainty；
   - observation aleatoric noise；
   - non-stationary process uncertainty。
2. 保留 `PredictionError` 四轴原始值和 credit 派生不变；uncertainty 只能先作为 owner-local plasticity evidence。
3. 建立 calibration evidence：Gaussian NLL / CRPS、80%/95% predictive interval coverage、covariance collapse/explosion、change-point recovery、old-regime revisit retention。

#### Phase 1：PE owner 内部的小型 Bayesian/Kalman predictive head（首选落点）

1. 在现有 world/self predictive heads 旁新增 SHADOW candidate：
   - 复用当前 compact evidence 特征与下一轮 realized outcome；
   - 每个轴维护低维权重 posterior mean + covariance；
   - 上一轮 posterior 作为下一轮 prior；
   - 原有 online-SGD head 仍是唯一 live writer。
2. candidate 只发布 owner 自己生成的 machine-readable calibration / latency / drift-recovery evidence；consumer 不读取内部 covariance，不创建第二 prediction owner。
3. 至少在 `>= 500 turn` 真 trace 上满足：
   - RMSE 相比现有 SGD head 改善 `>= 0.02`，且 NLL / CRPS 同向改善；
   - 80%/95% coverage 接近名义覆盖率；
   - change-point 后恢复速度更快，回访旧 regime 时不显著遗忘；
   - covariance 不持续塌缩到 0 或无界膨胀；
   - determinism、checkpoint/restore、rollback drill 与延迟门槛通过。

#### Phase 2：证据通过后，才把 precision evidence 接入 CMS

1. 将 calibrated gain 作为 `LearnedUpdateRule` 的附加 typed feature 或 owner-internal multiplier，例如：

```text
effective_step = learned_step * bounded_precision_gain
```

2. 它只调节"当前证据吸收多少"，不替代 ATLAS replay、Titans PE gate、slow→fast backflow、ModificationGate 或 rollback。
3. 先 SHADOW 对比 `new_knowledge_absorption / old_knowledge_retention`，通过长期 matched control 后才允许 ACTIVE。

### 明确不接入的地方

- **Temporal SSL 暂不直接接 Kalman gain**：其 KL 已使用 prior/posterior 方差约束；`z_t` 是抽象动作代码，不是环境 belief。直接叠加 gain 会混淆 latent uncertainty、parameter uncertainty 与 observation noise。
- **Credit 不做 inverse-variance 改写**：raw PE / credit ledger 必须保留真实发生过的误差；uncertainty 可控制学习速度，不能抹掉高噪声但真实的关系破裂证据。
- **`self_mod.py` / substrate 不接**：默认基底冻结；adapter 参数规模大，缺乏合法线性观测模型，维护 covariance 的成本与风险不成立。
- **不新增全局 Bayesian memory owner**：任何 posterior updater 只能是对应 owner 的内部算法，跨模块继续只走 immutable snapshot。

### 停止条件

- Bayesian head 只表现得更平滑，但 NLL、coverage、change-point recovery 不优于现有 SGD baseline。
- `R_obs` 只能靠手填常量才能工作，无法从 typed outcome 稳定估计。
- self/relationship 轨迹出现 posterior 过度自信，对修复证据反应迟钝。
- 为了传 covariance 而新增跨 owner 私有状态依赖，或让 consumer 重建 producer uncertainty。
- SHADOW 证据需要直接改变 live prediction / credit / temporal / substrate 才能显示收益。

### 预期收益

- 把不确定性从观测面升级为有校准证据的可塑性控制，同时保持 R-PE 原始信号不被改写。
- 在非平稳开放对话中更有原则地平衡"快速适应"与"抗遗忘"。

3. **WiringLevel 三态迁移**：新逻辑先 SHADOW 模式与旧逻辑并跑，对比 snapshot；通过后切 ACTIVE，旧逻辑保留为 DISABLED 直到一段稳定期。
4. **评估证据**：每条改动都需要在 [`docs/specs/evaluation.md`](../specs/evaluation.md) 的相关 family 上给出可对照、可 counterfactual 的 evidence；evidence 不达标 → 不切 ACTIVE。
5. **回滚窗口**：每条 ACTIVE 后保留 ≥1 个 release cycle 的回滚开关。
6. **OA 组特殊条款**：所有 OA-* 改动的 evidence 必须包含"对应外部论文描述的攻击 / 失败模式在我们这里复现率"——例如 OA-3 的 evidence 必须包含"重现 N4 的 5 类 misalignment 场景，验证 FramingAwarenessCheck 缓解率 ≥ 75%"。
7. **CMA 组特殊条款**：所有 CMA-* 改动必须先证明 read-only evidence；CMA-1 禁止直接写 durable memory，必须先以 SHADOW / telemetry 形式与 CMA-2 的 VZ-MemProbe 绑定验证。
8. **EVO 组特殊条款**：所有 EVO-* 改动必须 **不把演化回路接到 token 空间的长期策略学习或基底 fine-tune**（R2/R4）；LLM judge **仅 readout**；archive / scenario / meta-prompt 变异 **停留在 expression 层或 owner-internal proposal schema**，且遵守 R8 snapshot SSOT。
9. **COG 组特殊条款**：所有 COG-* 改动必须把社会心智 / persona / environment / interpretability 结果落到 typed owner、snapshot 或 read-only evidence；禁止用 prompt 标签、关键词规则、LLM judge reward、直接 activation steering 替代 owner 学习与 ModificationGate。
10. **SYS 组特殊条款**：所有 SYS-* 改动必须严格遵守 R2（冻结基底）和 R4（内部控制不在 token 空间）。KV Cache Steering 必须是单向且无梯度的；Latent Action RL 必须在独立的轻量级控制器中进行；A-Mem Schema 引入仅限数据结构，禁止引入不受控的 LLM-as-curator 图谱突变。
11. **BAYES 组特殊条款**：必须先定义并校准 `P`（epistemic）、`R`（aleatoric observation noise）、`Q`（non-stationary process uncertainty）；首个 packet 只能在 PE owner 内 SHADOW，禁止直接改 live credit、temporal、CMS 或 substrate。任何 covariance 都是 owner-private，consumer 只能读取 owner 发布的 compact calibration snapshot/readout。

### 与既有 [`real-open-dialogue-learning-loop.md`](real-open-dialogue-learning-loop.md) 的关系

本文件给的是"**研究借鉴的方向**"——基于学术外圈论文给的设计候选；
[`real-open-dialogue-learning-loop.md`](real-open-dialogue-learning-loop.md) 给的是"**真实 open dialogue 数据闭环工程**"——基于我们当前 runtime 的工程缺口。

两者关系：

- DM 组的 **DM-1 / DM-3 / DM-4 / DM-7** 都是 real-open-dialogue 大工程闭环里的**关键能力组件**。
- OA 组的 **OA-1 / OA-2 / OA-3 / OA-4 / OA-6** 都是 real-open-dialogue 大工程闭环必须配套的**安全护栏**——开放对话学习 = 系统从真实用户经验中持续适应，**没有这些护栏就是把 N4 描述的 production RL emergent misalignment 主动请进 VZ**。
- CMA 组的 **CMA-2 / CMA-3** 是 real-open-dialogue 大工程闭环里的**记忆证明与漂移监控**底座——开放对话学习会不断写入、晋升、整合真实用户经验，没有长时间窗 probe 和 drift telemetry，就无法证明 R5/R6 没有悄悄漂移。
- EVO 组的 **EVO-2 / EVO-3 / EVO-6（含 DM-7）** 是 open dialogue **评估与压力分布可演化**的基础设施——没有 **cascade eval** 与 **代际 head-to-head**，开放学习下的 ModificationGate 与场景库会退化为 **启发式赌博**；没有 **QD scenario archive**，压力测试 coverage 无法随系统弱点演化。
- COG 组的 **COG-1 / COG-2 / COG-3 / COG-11** 是 open dialogue 从"会学习的 assistant"走向"有社会心智的 digital life"的补强底座——没有反事实信用、ToM owner、persona/regime 几何监控与关系/EQ 长程评估，开放学习会缺少"为什么学、学到谁身上、身份是否仍稳定、关系是否真的变好"这四类证据。
- SYS 组的 **SYS-1 / SYS-2 / SYS-5** 是 open dialogue 闭环的底层算法骨架——没有 CPD 切换、VC 容量双门和 Latent Action RL 基线，高层的"涌现"和"自修改"将缺乏坚实的工程落脚点。
- BAYES 组的 **BAYES-1** 是 open dialogue 在线适应的 plasticity calibration 层——先回答"这条反馈有多可信、旧 belief 有多不确定、环境是否已变"，再决定吸收幅度；它不替代 PE、credit、CMS 或 ModificationGate。
- 建议在 real-open-dialogue 的 packet 推进过程中**主动 cross-reference 本文件的 DM/OA/CMA/EVO/COG/SYS/BAYES 七组**，让"能力扩展"、"安全治理"、"记忆证明"、"门控内搜索工程"、"社会心智 / 身份 / 信用链"、"底层架构补强"与"校准不确定性驱动可塑性"同步落地，避免"先开放学习再补护栏 / 再补评估 / 再补 archive / 再补社会心智"的危险次序。

### 总成本估计（SYS 组）

- **P0（本季度）**：SYS-1 + SYS-2 + SYS-5 ≈ S + M + L ≈ 6-12 周 / 8-20 PR。SYS-5 (Latent Action RL) 是重头戏，需要专门的架构预研。
- **P1（本年度）**：SYS-3 + SYS-4 ≈ M + M ≈ 4-8 周 / 6-15 PR。
- **P2（长线）**：SYS-6 ≈ L ≈ 8+ 周。涉及全局架构调整，需谨慎推进。

### 总成本估计（OA 组）

- **P0（本季度）**：OA-1 + OA-2 + OA-3 = S + S + M ≈ 5-9 周 / 6-12 PR
- **P1（本年度）**：OA-4 + OA-5 + OA-6 + OA-7 = 4 个 M ≈ 14-25 周 / 12-32 PR
- **P2（长线）**：OA-8 + OA-9 + OA-10 = S + L + S ≈ 8-15 周 / 10-18 PR

DM 组工作量未在原始来源中估计，但根据涉及 spec 的复杂度可粗估：DM-1/3/6/7 各 M 量级，DM-2/5 各 L 量级，DM-4 是 spec 改动 + 评估配套约 M 量级。

### 总成本估计（EVO 组）

- **P0（本季度）**：EVO-2 + EVO-6（常与 DM-7 合并 packet）≈ **M + S**，依赖 evaluation / gate spec 起草与契约测试骨架。
- **P1（本年度）**：EVO-1 + EVO-3 ≈ **2× M**，依赖 DM-7 / EVO-2 已有收敛。
- **P2（长线）**：EVO-4 + EVO-5 ≈ **S + M**，依赖 expression 隔离（OA-2）与 scenario archive（EVO-3）。

EVO 组不是新的「基底训练」方向，而是 **R10/R12 的工程加固层**：优先保证 **EVO-2（cascade）** 与 **DM-4** 同频落地。

### 总成本估计（CMA 组）

- **P1（本年度）**：CMA-2 + CMA-3-lite = M + S/M ≈ 4-8 周 / 4-8 PR
- **P2（长线）**：CMA-1 = M-L ≈ 4-10 周 / 4-10 PR

CMA 组不是新的主架构方向，而是 R5/R6 的**证明面 + 受控实验**：先做 CMA-2 评估，再做 CMA-3 telemetry，最后才考虑 CMA-1 shadow path。不要反过来先上 active memory graph。

### 总成本估计（COG 组）

- **P0（本季度 / 半年内起跑）**：COG-1 + COG-2 + COG-3 ≈ **M + M + S/M**，依赖 credit / semantic owner / regime spec 的现状核查与 SHADOW evidence。
- **P1（本年度）**：COG-4 + COG-5 + COG-6 + COG-7 + COG-8 + COG-10 + COG-11 ≈ **6× M + S/M**，多数可先作为 spec + read-only evaluation packet 推进。
- **P2（长线）**：COG-9 ≈ **M**，依赖 lifeform-vitals / cognitive-regime / character bootstrap 的成熟度与真实 open dialogue evidence。

COG 组不是新的"能力堆叠"，而是 **cognitive AGI / digital life 的社会心智与身份连续性补强层**：优先做 COG-1/2/3/11，先把信用、ToM、身份漂移、关系质量四个证据面补上。

### 总成本估计（BAYES 组）

- **P1（本年度）**：BAYES-1 Phase 0 + Phase 1 ≈ **M**，约 2-6 周 / 3-8 PR；只做 PE owner 内部 predictive-head SHADOW、校准评估与 rollback evidence。
- **后续可选**：Phase 2 CMS precision feature ≈ **S-M**；必须等待 `>= 500 turn` 真 trace 证明 Bayesian head 相比 SGD 的 NLL / coverage / change-point recovery 增益。

BAYES 组不是引入完整 Kalman Bayesian Transformer，而是把**经过校准的不确定性变成 owner-local plasticity evidence**。没有 `P/R/Q` 语义和校准证据时，不得因公式看起来更有原则就接入 live update。

### 资源紧张时最高 ROI 的 13 项

如果只能挑 13 项，按"防御性 / 直接保护现有不变量 + 底层机制刚需 + 立即可工程化 + 记忆证明面 + 认知生命体补强 + 评估 cascade + 校准可塑性"排序：

1. **OA-1**（P0/S）— 把 N3/N4/N6 写进 spec motivation，立刻让规则有外部实证背书。
2. **OA-2**（P0/S）— Mind/Face 隔离形式化，把 R4 升级为可机器验证。
3. **OA-3**（P0/M）— FramingAwarenessCheck，把 N4 inoculation 75-90% 缓解率工程化。
4. **DM-4**（与 OA-3 同期）— ModificationGate eval 前置门槛，把 R10 锁定。
5. **SYS-2**（P0/M）— VC 容量双门：与 DM-4 配合，彻底锁死 ModificationGate 的泛化安全。
6. **SYS-1**（P0/S）— CPD 涌现切换：用 PE spike 解决 $\beta_t$ 切换的无监督难题。
7. **COG-1**（P0/M）— 反事实信用分配 + least-control：让系统知道长期关系结果"为什么"发生。
8. **COG-3**（P0/S-M）— Persona / Regime Geometry 漂移监控：把 R14 身份稳定做成 read-only evidence。
9. **EVO-2**（P0/M）— evaluation **cascade**（cheap→expensive）+ LLM-judge **仅 readout**：把 DM-4 从"口号"变成**可执行验收管线**，直接加固 R12。
10. **BAYES-1**（P1/M）— Precision-aware plasticity SHADOW：先证明校准 uncertainty 能比固定/learned scalar gate 更好地平衡新知吸收与旧知保持。
11. **CMA-2**（P1/M）— VZ-MemProbe，把 R5/R6 的长期记忆连续性做成可重复证据。
12. **COG-2**（P0/M）— Social Cognition / ToM owner 学习：让多主体 belief / intent / feeling / preference 进入 owner 路线，而不是 prompt 推断。
13. **OA-4**（P1/M）— VZ-Audit Agent，把 N8 工程范式吸收为 R10 标配。

这 13 项形成"**安全治理 + 底层控制机制 + 反事实信用 + 身份漂移监控 + 记忆证明底座 + 评估完备性工程化 + 校准不确定性驱动可塑性**"；剩余 31 项可视具体季度优先级排队，均建立在这个底座之上。
