# 关系动力学：通用规律 vs 个体轨迹

> Status: design cognition note（认知基线，非实现 spec）
> Date: 2026-07-18
> 来源: DINQ BP 对照分析 → OSS 发布方案讨论中沉淀的核心认知
> 下游: `docs/specs/oss-relationship-representation-standard.md`（Option A）、
> `docs/specs/oss-relationship-encoder.md`（Option B）、
> `docs/business/BP/volvence-bp-sales-wedge-v1-cn.md`（护城河叙事 P4/P9）
> 对应需求: R2（稳定基底 + 自适应控制器）、R8（快照 SSOT）、R12（评估只读）、R15（证据先行）

## 0. 一句话

**关系轨迹因人而异，关系动力学跨人共享；轨迹是数据和状态，动力学才是模型。
开源通用动力学，闭源个体适应与关系状态——切口正好落在 R2 的基底/控制器分层线上。**

## 1. 要回答的问题

在规划「开源人类/关系表征模型」时出现的两个根本问题：

1. 我们应该输出的是**基于具体某个人**（如张无忌）的关系表示，还是**通用的**关系权重模型？
2. 如果是通用模型——关系轨迹难道不会因人而异吗？通用权重凭什么成立？

## 2. 两层结构：轨迹是状态，动力学是规律

### 2.1 "与某个人的这段关系"不是权重，是运行时状态

一段具体关系在系统里的存在形态是：记忆写入、9 类 semantic owner 的 snapshot、
rupture/repair 轨迹、信任演化序列。它有三个属性决定它**不能也不该**成为开源权重：

1. **撑不起"模型"**：单条轨迹没有泛化面，只是一份档案；
2. **是隐私资产和商业资产**：per-tenant scope，正是飞轮里"客户付钱帮我们积累"的部分，
   开出去等于把关系资产本身送人；
3. **每天在变**：权重形态（冻结、慢更新）装不下高频演化的状态。

### 2.2 通用模型学的是所有关系共有的"物理规律"

信任如何建立与流失、修复窗口长什么样、承诺的兑现与违约如何影响后续、
什么信号预示关系升层或降层——这些**转移规律**跨人、跨 persona、跨场景成立，
才值得压进权重、才配叫 foundation model。

类比物理：每个抛出去的球轨迹都不同（初速度、角度、质量各异），但所有球服从同一组方程。
模型学的不是"标准轨迹长什么样"，而是**从已观测交互历史到关系状态的映射**。

类比心电图：每颗心脏都不一样，但"心律失常长什么样"跨人泛化——
前提是模型看的是**这个人自己的信号**。

### 2.3 个体性通过输入进入判断，不压进权重

encoder 是条件模型：读**这个人**的轨迹，输出**这段关系**的状态。

- 张三被冒犯后三天缓和，李四记恨三个月——两条轨迹完全不同；
- 但"修复尝试出现在信任低点之后、且被对方接受，关系才解锁"这条转移规律对两人都成立，
  只是参数不同。

个体差异不是通用模型的敌人，而是它的**输入**。

## 3. 诚实边界：两阶个体差异，通用模型只覆盖一阶半

| 阶 | 差异内容 | 通用模型能否覆盖 |
|---|---|---|
| **一阶** | 历史不同、状态不同（同规律、不同初始条件） | **完全覆盖**——以轨迹为条件即可 |
| **二阶（可推断部分）** | 转移规律本身因人而异（修复速度、边界敏感度、文化规范），但轨迹内有证据 | **摊销推断覆盖**——"此人历史上三次修复都花了很久"→ 调低本次修复预期；轨迹够长时通用权重做得到 |
| **二阶（不可推断部分）** | 轨迹内看不出来的个体规律 | **不可约误差**——通用模型的硬边界 |

## 4. 不可约误差恰好是商业运行时的地盘（R2 投影）

| 层 | 内容 | 开源边界 |
|---|---|---|
| **冻结基底** | 跨人共享的关系动力学 | **可开源**——对谁都一样，开出去不损失个体资产 |
| **自适应控制器 + per-user 状态** | 每段关系的个体偏差（在线适应、每实例、私有） | **闭源**——这就是付费产品本身 |

反证两个极端，说明产品为什么必须存在于中间：

- 若关系规律**完全通用** → 一个静态模型就够，运行时没有存在必要；
- 若关系规律**完全因人而异** → 任何模型都帮不上忙，冷启动不可能，产品也不成立。

真实世界在中间：**通用模型提供先验和冷启动，个体适应吃掉剩下的方差。**

护城河推论（写入销售楔子 BP 的叙事基础）：竞品拿到开源 encoder 也只拿到先验；
每段关系的个体偏差，仍然只在**持续经营这段关系的运行时**手里。
开源通用部分不削弱护城河，反而把"接入标准"的入口做大。

## 5. 具体人物（张无忌类）的两个正确角色

都不是"发布一个张无忌关系模型"：

1. **训练数据的生成载体**：live-through 管线让角色"活过"一部叙事，
   产出带 ground-truth 的长程关系轨迹（张无忌×赵敏 / ×周芷若 / ×谢逊 各是一条样本）。
   通用 encoder 需要成千上万条这样的轨迹，具体人物只是其中一族。
   （版权约束：开源管线分支只用公版/自造语料，现有受版权语料不外流。）
2. **标准的 demo 实例**：发布时附可复现示例——"用这条公开轨迹跑 encoder，
   得到这样的关系状态读数"。发布物是管线和示例轨迹，不是某个人的关系权重。

## 6. 自检测试

> **"如果明天来一个新用户/新角色，这个发布物对他有用吗？"**

- 通用 encoder：有用——直接对他的轨迹出读数。→ 这是模型，可开源。
- 某段关系的档案：没用——那是别人的关系。→ 这是数据/状态，不开源。

具体关系状态在商业侧的正确形态是既有产品：**Relationship Sales Map**（付费交付）与
**Relationship Passport**（用户可治理的资产）——永远不进权重、不出租户。

## 7. 可证伪性：跨人泛化是经验问题，不是信念问题

"通用动力学到底成立多少"由实验回答，不需要预先相信：

- **实验设计**：held-out 场景族**整族隔离**——评测用训练中从未见过的 persona 族，
  看结构化预测是否仍显著优于 LLM zero-shot 对照（Option B spec 的 G2 gate）。
- **证伪后果**：若跨 persona 泛化垮掉，即证伪"通用动力学"前提，
  按 spec 止损路径降级（数据管线降级为 Option A 的 conformance 数据生成器），
  损失有界。
- **对应纪律**：R12（评估只读，benchmark 不被蒸馏进权重）+ R15（证据先行，
  gate 未过权重不出仓）。

## 8. 认知固化为三条不变量（供 spec 与 BP 引用）

1. **权重只装跨人共享的动力学；一切个体性经输入（轨迹条件）或运行时状态（owner snapshot）承载。**
   任何"把某个具体人/具体关系压进权重"的提案违反本条。
2. **开源/闭源切口 = R2 基底/控制器切口。** 通用动力学（慢、冻结）可开；
   个体适应与关系状态（快、每实例）不开。切口移动必须先改本文档并给出证据。
3. **通用性主张必须可证伪。** 对外宣称"关系动力学模型"之前，
   跨 persona 泛化必须在整族隔离的 held-out 上过 G2；未过则不使用该措辞。

## 9. 词位注记：「人类表征模型体系」与本文不矛盾

公司叙事层（国内资本场合）主词定稿为「人类表征模型体系」，与本文的分层论证兼容，理由是：
**人是表征对象，关系是表征方法与数据通道**。生命体系（记忆 / user_model / ToM /
需求状态 / 边界共识 / 关系状态）表征的正是「关系中的人」——区别于 People API 类公司
从公开痕迹表征「社会中的人」。「体系」二字同时名实相符（schema + 运行时 + 记忆 +
未来 encoder），内建规避「大模型」的权重透支与备案语境。

边界保持不变：

- 本文三条不变量约束的是**开源边界与权重内容**（权重只装跨人共享动力学），
  不约束公司叙事用词；
- 「人类表征大模型」字样封存：Option B 权重落地并过 G2 前任何场合不使用，届时再评估升级；
- 英文场合不使用 DINQ 英文原词，用 relationship representations / Relationship Runtime
  （使用规则的 SSOT 在销售楔子 BP P12）。
