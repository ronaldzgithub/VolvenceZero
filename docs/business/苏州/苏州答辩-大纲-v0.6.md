# 第 5 页 — 市场情况及痛点

> 模板要求：抛出行业 / 市场痛点、传统方案；描述客户、市场和问题；目前客户如何应对；强调问题严重性及竞品无法满足的客观事实。

- 核心判断：大语言模型解决了「智商」（知识、推理），但**无法建模「关系」**——而关系恰恰是情感陪伴和长程任务里最关键、最难的部分。
- 为什么做不到：现有 AI 应用都在靠「上下文工程」（把记忆、人设、规则塞进 prompt）凑合，但关系是动态演化的——**无论情感陪伴还是长程任务，靠拼上下文都难以实现**。
- 具体痛点①育儿顾问：育儿有无数流派（亲密育儿 / 行为训练 / 蒙台梭利…）——**到底听谁的？** 必须结合这个家庭、这个孩子、这段关系的历史动态判断，而不是背一个知识库。
- 具体痛点②私域销售：一个真人金牌销售**无法一对 2000**；而要同时具备领域知识 + 世界知识、还会**高情商聊天**的人，既贵、又难招、还易流失、会带走客户。
- 结论（要解决的核心）：知识可以靠大模型补齐，真正缺的、必须解决的是**情商**——理解关系、懂分寸、判断时机。这就是我们要攻克的核心问题。

**主讲话术：** 今天大模型很聪明，但它解决的是「智商」。情感陪伴也好、长程任务也好，真正难的是「关系」——而关系是动态的，靠把上下文塞进 prompt 是建模不出来的。比如育儿顾问，育儿流派那么多，到底听谁的？得看这个家庭、这个孩子、这段关系怎么走。再比如私域销售，一个真人没法一对 2000，而又懂行业、又懂世界、又会高情商聊天的人，太贵也留不住。所以我们要解决的核心，不是知识，是情商。

**对应评分：** 项目重要性；产业带动；需求迫切性。

# 第 7 页 — 核心技术

> 模板要求：核心技术 + 知识产权布局，讲清先进性、创新点、差异化与壁垒，可提专利 / 知识产权。
> 本页主线：以杨柳博士二十年的学术体系（人机交互复杂度 + 主动学习 + 强化学习 + 迁移学习）为理论根基，工程化为数字生命的三大自主能力——**主动学徒式学习、认知迁移、关系动力模型**。

**一、理论根基：杨柳博士的学术体系（世界顶尖）**

- 出身与方向：CMU 机器学习博士（该系人工智能世界排名第一），博士论文《Mathematical Theories of Interaction with Oracles》，研究人机交互的复杂度；答辩委员会成员含图灵奖得主 Manuel Blum。
- 核心判断：通用人工智能的重要组件是强化学习，但强化学习有两个瓶颈——①训练所需标记数庞大；②泛化能力差。我们二十年的研究正对准这两个瓶颈：用**主动学习**大幅减少所需标记数，用**迁移学习**提升泛化能力并提供**数学保证**。
- 学术体量：在主动学习、强化学习、迁移学习领域发表顶会顶刊论文 30 余篇（全部 CCF-A 类顶会 / 顶刊 40 余篇，备清单佐证）。

**二、主动学习（破解「标记数庞大」瓶颈）— 三大代表作**

1. Surrogate Losses in Passive and Active Learning：针对 0-1 loss 不可高效优化的问题，提出 surrogate convex loss，解决 efficiency issue；发表于统计学顶级期刊 *Electronic Journal of Statistics*。
2. Active Learning with Identifiable Mixture Assumptions：解决主动学习领域一个长期待解决的问题；正投往统计学顶刊 *Annals of Statistics*。
3. Minimax Analysis of Active Learning：提出复杂度度量 **Star Number**，自提出起被工业界与学术界沿用为标准；发表于机器学习顶刊 *JMLR*。

→ 工程化落地（**主动学徒式在线进化**）：数字生命不会做时主动向人类专家请教，用更少标注学习，持续训练数据量压缩至传统方案的 1/100，实现实时在线进化。

**三、主动学习 × 强化学习（让数字生命「会自主、懂求助」）— 三大近期代表作**

1. Bandit Learnability can be Undecidable：在 Nature《Learnability can be undecidable》提出 EMX problem 的基础上，构造从 bandit learning 到 EMX 的 reduction，证明 bandit 可学习性可不可判定；发表于机器学习理论顶级会议 **COLT 2023**。
2. Reliable Active Apprenticeship Learning：火星车类比——尽量自主探索，但快跌下悬崖时及时向人类求助、且求助次数尽量少（地火通信昂贵且有时延）；知道最优动作就执行，不知道就向人请教、从调教中重训参数；发表于 ALT 2025。
3. Active RLHF：RLHF 中训练 reward model 需大量标记后再交给 PPO；我们的主动学习算法可大幅减少训练 reward model 所需的标记数。

→ 工程化落地：数字销售 / 数字生命「不会就问、问得最少、问完会学」，是低成本持续进化与人审可控的算法基础。

**四、迁移学习（破解「泛化差」瓶颈）→ 认知迁移**

- Idea：会了英语，学法语更快。算法：从过去任务数据中估计一个**贝叶斯后验分布**；后验估计不准时 run prior-independent 算法，后验估计较准时 run prior-dependent 算法。
- 数学保证：当任务数趋于无穷，每个任务所需样本数的上限 =（提前已知先验时每任务所需样本数）+ VC 维度——在事先并不知道先验分布的前提下，这是一个很强的保证。
- 与强化学习中的元强化学习（meta-RL）思路一致；未来研究方向：为元强化学习提供数学保证。
- → 工程化落地（**认知迁移**）：把专家的经验、判断与分寸（情商）连同知识与能力（智商）一起迁移进数字生命；学会一个客户 / 场景后，能少样本快速迁移到新客户 / 新场景，跨任务复用经验。

**五、自主模型：关系动力模型（直接对应第 5 页「关系」痛点）**

- 大模型解决「智商」，但关系是动态演化的，是情商的核心。我们的**关系动力模型**显式建模「关系随时间的演化动力」——记住关系历史、理解推拉与信任、把握分寸与成交 / 介入时机，而不是靠上下文工程往 prompt 里硬塞。
- 三层合一：主动学徒式学习（少标注 + 自主 / 求助）+ 认知迁移（跨场景泛化）+ 关系动力模型（动态关系建模），构成数字生命「人格一致 + 持续进化 + 懂关系」的自主技术栈，是我们最难被复制的壁垒。

**六、支撑工程与知识产权**

- 人格一致性类脑 MOE 架构：围绕长期记忆、角色目标、情感表达、任务策略，实现稳定独立人格与情感涌现，解决「人设崩塌、无长期记忆」痛点。
- 全链路运行时 Volvence Runtime：中枢编排、情境感知、决策核心、情感计算协同，支持端侧轻量化离线部署，响应延迟 ≤200ms，解决隐私、算力、网络依赖三大难题。
- 低代码定制化引擎：拖拽式完成性格塑造、能力配置、记忆植入、外观定制，大幅降低场景适配门槛，实现标准化、规模化复制。
- 知识产权布局：核心技术完全自主；学术成果已被 NeurIPS 2026、AAAI 2027 接收，ALT 2025 已发表；计划发明专利受理 6 项、PCT 受理 3 项。
- 研究团队：杨柳 + Steve Hanneke（普渡大学计算机系助理教授、主动学习理论奠基人、COLT 2026 主席）+ CEO 赵江波；二十余年累计在三个领域顶会顶刊 30 余篇，是世界顶尖的主动学习研究团队。

**主讲话术：** 我们的核心技术不是堆参数，而是有二十年学术体系支撑。通用人工智能最关键的是强化学习，但它有两个瓶颈：训练要的标注太多、泛化太差。我用主动学习解决第一个——让系统不会的时候主动请教人类，用极少标注就能学，把持续训练数据压到传统方案的百分之一；我用迁移学习解决第二个——像学会英语再学法语一样，把一个客户、一个场景学到的能力迁移到新客户、新场景，而且有数学保证。在这之上我们做了关系动力模型，专门建模「关系」随时间的动态演化——这正是大模型做不到、又是情感陪伴和长程销售最值钱的能力。「主动学徒式学习 + 认知迁移 + 关系动力模型」就是我们最难被复制的壁垒。
