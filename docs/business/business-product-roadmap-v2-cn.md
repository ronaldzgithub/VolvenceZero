## 0. 一句话

我们做一层**关系控制中间层**：把任意大模型的"一次性 IQ"，变成可长期记忆、懂分寸、能被监管和接管的"关系型 EQ + IQ"。
它落到两个业务方向、一个变现结构：

- **ToB —— 数字员工**：企业雇一个能上岗、考核、监管、接管的 AI 员工。
- **ToC —— 陪伴电商**：在母婴/私域/IP 场景养长期陪伴的 AI 运营员工，沉淀关系电商资产。
- **结构**：**合资/合作为主，自营为辅**——合作方出流量与履约，我们出"进化涌现引擎 + 治理 + token"，自营握住直接关系与证明场。

---

## 1. 为什么是这两个方向（业务判断）


|               | 别人在做什么                    | 我们的空白               |
| ------------- | ------------------------- | ------------------- |
| 基础大模型公司       | 卷通用 IQ、通用助手               | 不会为每个垂直关系单独建长期记忆与治理 |
| 垂直 App / SaaS | 拼 prompt / RAG / workflow | 撑不起长期关系，也做不出可审计可接管  |
| MCN / 品牌 / 平台 | 有流量、有粉丝、有渠道               | 没有"能长期经营关系的人"       |


**结论**：长期关系型产品都需要、但没人做好的那一层中间层，正是我们的位置。两个方向是这层中间层的两种售卖姿势——**对企业卖"员工"，对消费场卖"陪伴"**，底座同一套。

## 2. 业务方向一：ToB —— 数字员工

### 2.1 是什么

不是 chatbot，而是一个**会上岗的 AI 员工**：有长期记忆、关系状态、主动学习、内部决策、审计接管、持续身份。客户像招人一样"领养 → 培训 → 考试发证 → 上岗 → 监管"。
同一套底座可包成不同**职能岗位**——客服/选品助手、企业知识官，也包括**数字 IP / IP 网红分身、AI 销售员、AI 市场推广运营**，差别只在运营手册与考核标准。

### 2.2 卖给谁 / 解决什么


| 客户              | 场景                    | 用它做什么                                   |
| --------------- | --------------------- | --------------------------------------- |
| 跨境平台 / 出海企业     | 恒一、浙江数字国贸大可平台、1688 生态 | 多语言、合规、可私有化的 AI 业务员/客服/选品助手             |
| 行业大模型方          | 雷神 IAQ-Brain（室内空气安全）  | 把垂直知识做成可治理的混合大模型服务                      |
| 企业知识官           | 内训、客户成功、销售辅助          | 一个 AI 跨会话服务多员工/客户，记忆不丢                  |
| 品牌 / MCN / IP 方 | 数字 IP、IP 网红分身         | 用 AI 长期承载 IP 人格，规模化经营内容与粉丝关系（不依赖真人随时在场） |
| 销售型企业           | 获客、跟单、成交转化            | AI 销售员跨会话记住每个客户，主动跟进、推进商机、可考核可监管        |
| 市场 / 增长团队       | 市场推广、活动运营、私域增长        | AI 做内容投放、线索培育、活动陪聊，把曝光沉淀成可经营的长期关系       |


### 2.3 产品形态（由轻到重）

1. **SaaS**：开箱即用，按账号/对话量计费——中小客户主入口。
2. **私有化 / VPC**：模型+中间层独立部署，数据不出域——中大型企业与合规场景。
3. **JV / 联合品牌**：与平台共建产品、收益分成——战略级客户。

### 2.4 上岗 5 步（统一交付动作）

领养 AI → 配置运营手册 → 主动自学 → 考试发证 → 上岗运营。
配 4 个面板：训练 / 运营 / 审计 / 关系。**护城河在"审计·接管·可删除·可回滚"——企业敢采购的前提。**

## 3. 业务方向二：ToC —— 陪伴电商

### 3.1 是什么

私域真正缺的不是工具，是"人"——一个有泛智商、领域智商、情商、主动学习、还愿意花长期时间的运营。我们交付的就是这个**会成长、可考核、可监管的 AI 运营员工**。

### 3.2 三个切入场景


| 场景      | 合作方                      | 关系价值                         |
| ------- | ------------------------ | ---------------------------- |
| 母婴全周期陪伴 | 高盖伦（千万级母婴粉）              | 0–18 岁亲子轨迹，长期 switching cost |
| 私域陪伴电商  | 摩比 / 小小莎 IP / 乔栋（千万级私域粉） | 把公域流量沉淀成可复购的私域关系             |
| IP 数字分身 | 唐商文化 / 名人 IP 矩阵          | 用 AI 长期承担 IP 的关系经营           |
| 自营轻订阅   | Mira（自营 C 端）             | 直接握住用户关系，验证付费意愿              |


### 3.3 产品演进路径（这是 ToC 的真正野心）

> 内容电商用内容建立注意力；货架电商用搜索承接需求；**我们做关系电商——用陪伴代替内容，用信任代替流量。**

电商的底层资产一直在迁移：从"流量"到"内容"，再到"搜索心智"。下一层、也是最难被平台收走的一层，是**长期信任关系**。内容会过气、流量会涨价、搜索心智会被巨头垄断，但一个用户愿意长期托付、持续复购、主动倾诉的"关系"是非标、专属、且越用越深的——这正是我们要沉淀的资产。我们的演进路径，就是沿着这条关系资产链由近及远地铺开：

**第一阶段·私域运营（现在）——把"会经营关系的人"先跑通**
在已有私域流量池里上岗 AI 运营员工：日常陪聊、答疑、推荐、复购提醒、关键时刻交人工。目标不是冲一时 GMV，而是验证一件事——**关系连续性确实能换来更高的转化、复购与留存**。这一阶段我们握住的是"单个关系的经济价值"。

**第二阶段·公域转私域（向上：内容场陪聊答疑）——把陌生流量变成可经营的关系**
把 AI 运营推到内容场（直播间、评论区、内容种草的承接环节），用一对一陪聊和答疑，承接公域涌入的陌生注意力，并把它**沉淀为私域里有记忆、有关系状态的长期用户**。区别于传统"投流—收割"：我们不是把流量一次性变现，而是把它转化成可反复经营的关系资产，摊薄获客成本、拉长用户生命周期。

**第三阶段·关系驱动供给（向下：按长期需求组织货）——让关系反向定义货**
当 AI 长期陪伴沉淀了足够多真实、连续的需求轨迹（孩子几岁、皮肤状态、消费节奏、复购周期……），供给侧的逻辑就反过来了：**不再是"有什么货推什么"，而是"用户长期需要什么，就去组织什么货"**。关系数据成为选品、组货、甚至 C2M 反向定制的依据，毛利与匹配效率同步抬升——这是货架电商和内容电商都给不了的精准度。

**第四阶段·关系电商平台（长期：新交易入口）——从垂直工具升级为平台**
当跨垂直（母婴、私域、IP……）的长期关系图谱足够厚，这些"会经营关系的 AI 运营"和它们承载的信任关系，本身就构成一个**新的交易入口**：用户不再先搜商品、刷内容，而是先回到长期信任的关系里，由关系来组织交易。这一步是长期期权（不在三年框架内），但它定义了天花板——**我们最终要做的，是关系驱动的下一代电商基础设施。**

> 一句话串起来：**私域运营（现在）→ 公域转私域（向上：内容场陪聊答疑）→ 关系驱动供给（向下：按长期需求组织货）→ 关系电商平台（长期：新交易入口）**。每一步都在加深同一种资产——长期信任关系，而这正是别人最难抄、平台最难收走的东西。

### 3.4 为什么 prompt/RAG/workflow 不够

它们能写人设、查资料、编排步骤，但都答不了：**此刻该关心、该推荐、该停下，还是该交给真人?** 这恰恰是关系经营和转化的核心判断——只能靠会学习、有内部状态的运行时，不能靠规则堆。

## 4. 变现结构：合资为主、自营为辅（业务逻辑，不是财务比例）

### 4.1 三种模式

- **合资（主力）**：合作方出流量+履约，成立合资公司；合资公司向我司采购**进化涌现 token 服务费**（旱涝保收），我司持股拿**分红**（上行弹性）。代表：高盖伦、摩比私域、恒一。
- **合作分润 / AI 服务（补充）**：定制化、联合研发、出海 SaaS 项目分润。代表：浙江数字国贸、雷神。
- **自营（关键的少数）**：直接收订阅服务费。代表：Mira、 IP 分身。

### 4.2 为什么主力是合资（asset-light）

1. **借力买不起的资产**：千万级粉丝、跨境渠道、履约能力都在合作方手里；我们只出最稀缺的"会经营关系的中间层"。
2. **可复利**：同一套中间层承载多个合资公司，边际成本主要是算力，越铺越省。
3. **现金流健康**：token 服务费是水电费式稳定收入，分红提供上行——不背重资产获客。

### 4.3 为什么必须保留自营（约占小头、但不可砍）

1. **直接关系 + 第一方数据**：不经合作方截断的用户长期轨迹，是产品迭代和"关系即数据护城河"的源头。
2. **定价锚**：自营真实付费意愿，是与合资方谈 token 单价/分红比例的议价基准。
3. **证明场**：不受合作方约束的 benchmark / 上岗考核试验田，反哺 ToB 标准化交付。

> **比例定调（业务意图，量级与 excel 对齐）**：合资/合作长期占大头（约 3/4 强），自营占小头（约 1/4 弱）；**自营占比可被规模化摊薄，但绝对体量必须年年增长**——它是关系与数据的根，不是利润的根。

## 5. 产品路线：一套底座，两条产品线，四个阶段

### 5.1 同一底座

关系控制中间层 6 模块：长期记忆 / 关系状态 / 主动学习 / 内部决策 / 审计接管 / 持续身份。
ToB 把它包成"员工"，ToC 把它包成"陪伴"，**进化涌现 token 是统一的计费与能力单位**。

### 5.2 四个阶段（业务里程碑，节奏与 excel 三年框架一致）


| 阶段                            | 主线动作                                                        | 要证明的事                     |
| ----------------------------- | ----------------------------------------------------------- | ------------------------- |
| **Phase 1（首年）单位经济 + 关系资产证明**  | 私域跑通转化、母婴跑通长期轨迹、Mira 跑通自营订阅、恒一/浙江国贸 B2B2B 出海 SaaS 上线、雷神 PoC | 关系连续性确实带来转化/复购/留存提升       |
| **Phase 2（次年）平台规模化**          | token 引擎多合资公司复用，持续拓展新平台（B2B2C + B2B2B）滚动落地                  | 中间层可复用、可批量上岗，不是单点定制       |
| **Phase 3（第三年）公域转私域 + 治理护城河** | 公域→私域关系转化、关系驱动供给、企业级治理可审计/私有化成熟                             | 治理是真壁垒，关系开始反向组织供给         |
| **Phase 4（之后）关系电商平台**         | 跨垂直关系图谱，成为关系驱动的新交易入口                                        | 从垂直工具升级为平台（属长期期权，不在三年框架内） |


---

## 6. 边界：我们不做什么（反目标）

- 不训练基础大模型；站在大模型之上做中间层。
- 不自建流量、不自营履约重资产——守住 asset-light。
- 自营不与合资方抢同一渠道，避免内部左右手互搏。
- 不为冲短期规模牺牲治理/审计能力（那是 ToB 的采购前提，也是 ToC 合规上线的前提）。

---

## 7. 财务框架（仅作背景，不是测算）

> 这一版不做财务测算。仅说明业务节奏与 excel 三年框架量级一致：**首年起步、次年数量级跃升、第三年再翻数倍**，毛利与净利率随规模化与 token 复用逐年走高。具体逐行金额、单位经济、成本结构、kill criteria 见对账版 `business-product-roadmap-v1-cn.md` 与 `00：19-详细版本(1).xls`。

业务侧只需记住三句话：

1. **Phase 1 证明单位经济与关系资产**；
2. **Phase 2 靠平台化把同一底座复制到更多合资公司**；
3. **Phase 3 靠治理护城河 + 关系驱动供给，从工具走向平台。**
