**v2.1 修订要点**：(1) **2026 收入对外口径升至 3,800 万**（v2.0 原"内部经营目标 3,800–4,000 万"升级为对外口径），收入分配为**合资 2,165 / API 1,140 / 其他 495**（占比维持 57% / 30% / 13%）；运营支持成本随收入小幅同比例上调约 +180 至 **2,430 万**，2026 净利润由 357 → **477 万**，对协议下限 3,500 万**安全垫由"持平达标"升至 +300 万**；(2) 2027 API 同比由 +671% 微调为 **+611%**（基数 1,050 → 1,140，金额 8,100 不变）；(3) §1.3 / §2.x 新增 **API 增长逻辑专项注**——主要驱动 = **已有装机量消耗 + 网络级裂变传播**，不依赖等比例增加销售人头；(4) **新增独立精简版** [docs/business/支出计划-投资人版.md](支出计划-投资人版.md)，作为对外提交主稿；本文件保留为详细工作底稿。**2027 / 2028 收入、净利润、CapEx、编制 105 / 174 全部不变。**

## 一、总览：要达成什么、三年长什么样

### 1.1 与协议及战略主线对齐

本计划对齐增资协议及补充协议核心经营指标：**2026 年收入不低于 3,500 万元；2027 年净利润不低于 1,500 万元；2028 年净利润不低于 3,000 万元。**  

商业结构上强调三件事：**API / 平台类收入占收入比逐年明显提高**；**2026 年完成 12 家合资公司落地**并形成可复制交付；**融资各轮按「占有市场与规模化交付」的真实资源需求定规模**，保留安全垫，不为叙事抬升募资额。

### 1.2 三年核心数据一览

| 年度 | 实际收入 | 总支出（含折旧） | **会计净利润** | 当年新增 CapEx | 协议 / 规划硬约束 | API / 平台类占收入比 |
|------|--------:|---------------:|---------------:|---------------:|-------------------|---------------------:|
| 2026 | **3,800** | **3,323** | **477** | **250** | 收入 ≥ 3,500 万（+300 安全垫）；**12 家合资公司** | **约 30%** |
| 2027 | **18,000** | **15,417** | **2,583** | **4,000** | 净利润 ≥ 1,500 万 | **约 45%** |
| 2028 | **45,000** | **39,683** | **5,317** | **8,000** | 净利润 ≥ 3,000 万 | **约 60%** |

> **口径说明**：表中"总支出"含资产折旧（按 3 年直线计提，详见 **§四**），不含 CapEx 现金支出本身；"会计净利润"为会计口径，对应协议利润约束。**当年新增 CapEx** 为现金口径的资本性支出，单独通过 A/B 轮融资与经营回款支持，不重复进损益（详见 **§四**）。

**API / 平台类收入**（全文同口径）：含按调用量计费的 API、token / 平台服务费，及与平台化交付直接相关的技术服务；与股权分成、一次性项目费区分列示，便于逐年比较占比。

### 1.3 执行节奏与组织规模（提要）

- **2026 年**：商业化与大规模交付自 **5 月**起量，**人员扩张、算力与履约相关支出主要集中在 5–12 月（8 个月）**；损益仍按 **12 个月完整年度**列示。年末编制 **约 35 人**：**总部核心 15 人（研发 8 + 运营 4 + 管理 3）** + **生态专员 20 人（12 家合资公司驻场 12 人 + 总部带教 / 培训中台 / 客户成功 8 人，详见 §3.2）**；全年 **加权人年约 20–24**。  
- **2027–2028 年**：完整经营年度，年末编制分别约 **105 人、174 人**（职能口径见 **第三章**）；**收入增速主要由 API / 平台服务驱动**（API 占比 30% → 45% → 60%），合资公司家数 **2027 年累计 16 家、2028 年累计 20 家**——增速明显放缓，重在已签约存量达产**且单家年化贡献向 `合资公司资金收入测算.md` 稳态值收敛**，**人头增速远低于收入增速**。

**三年合资公司家数与 API / 平台收入对照**

| 年度 | 累计合资公司（家） | 同比新增 | 合资公司相关收入 | 单家年均贡献 | API / 平台服务收入 | API 同比 |
|------|-----------------:|--------:|----------------:|----------:|------------------:|--------:|
| 2026 | 12 | +12 | 2,165 | ≈ 180 | 1,140 | — |
| 2027 | **16** | +4 | 9,000 | ≈ **563** | **8,100** | **+611%** |
| 2028 | **20** | +4 | 16,000 | **800** | **27,000** | **+233%** |

> 2027 / 2028 收入扩张的**主引擎是 API / 平台**（同比 +611% / +233%），合资公司端转向**已签约存量的达产爬坡 + 少量精挑新增**；不依赖合资公司家数线性放大。

> **API 增长逻辑（v2.1 新增）**：API / 平台收入 2026 → 2027 +611%、2027 → 2028 +233% 的扩张**不依赖等比例放大销售队伍**，主要由两个网络效应驱动：
> (1) **已有装机量的消耗**：2026 完成 12 家合资公司落地后，每家本身即是大流量入口与产业链支点，其 C 端用户、合作伙伴、上下游客户的 API 调用量随业务自然消耗持续放大，无需重新获客；
> (2) **网络级裂变与传播**：合资 / 平台客户的成功案例形成口碑外溢与跨场景集成（SaaS 间互调 / 数据飞轮 / 内容生态），新客户接入边际成本递减；
> 这与 §3.2 编制结构一致——商务 / BD 自有人头占运营编制约 50%，**不为线性增长配等比例销售队伍**。

### 1.4 与协议条款逐条对照（总表）

| 条款 | 要求 | 本计划（会计口径） | 安全垫 |
|------|------|--------|--------|
| 补充协议收入 | 2026 年收入 ≥ 3,500 万 | **3,800 万** | **+300 万** |
| 补充协议利润 | 2027 年净利润 ≥ 1,500 万 | **2,583 万** | +1,083 万 |
| 补充协议利润 | 2028 年净利润 ≥ 3,000 万 | **5,317 万** | +2,317 万 |

> 协议利润口径为**会计净利润**（资产折旧入损益、CapEx 资本化）。本计划自 v1.6 起统一按该口径列示，对协议指标均留有安全垫；详见 **§四 资本性支出与折旧**。

> **2026 收入对外口径升级（v2.1）**：v2.0 中"内部经营目标 3,800–4,000 万"已**升级为对外口径 3,800 万**（合资 2,165 / API 1,140 / 其他 495），对协议下限 3,500 万留 **+300 万安全垫**（覆盖单笔合资延签 / 回款滞后约 ±8% 不确定性）；损益按 3,800 万对外列示，2026 净利润同步由 357 → **477 万**。

> **股份支付（期权摊销）口径（v2.0 新增）**：本计划损益**不含**股份支付费用。**2026 年期权池尚未启动，无影响**。**2027 年起**若审计要求按 IFRS 2 / CN GAAP 11 号将股份支付入损益（按 A 轮估值假设 5 亿、10% 期权池、4 年归属粗算约 **1,250 万 / 年**），2027 净利润将由 2,583 降至约 **1,333 万**（**贴近 1,500 协议下限**）；2028 含累计摊销将由 5,317 降至约 **2,192 万**（**可能击穿 3,000 协议下限**）。**强烈建议在补充协议中预先明确利润口径**——业内惯例以"扣非后净利润 + 加回股份支付"作为对赌口径，避免歧义触发条款。详见 §3.3。

### 2.1 2026 年：收入 3,800 万 · 会计净利润 477 万 · 12 家合资公司

**年度定位**：协议收入达标年（**收入 3,800 万 / 协议下限 3,500，+300 万安全垫**）；**12 家**合资落地 + **API / 平台约 30%** 基线；**会计口径净利润 477 万**（资产折旧入损益、CapEx 资本化，详见 **§四**）；经营放量与成本集中在 **5–12 月**（节奏见 **§3.1**）。

**收入结构**

| 来源 | 金额 | 占收入比 | 备注 |
|------|-----:|--------:|------|
| 合资公司股权 / 分成 / 联合商业化 | **2,165** | 约 57% | **12 家**，单家年均 ≈ 180 万（**含分红 + 股权 + 联合商业化 + 合资公司主体内部技术服务费分成**；爬坡 + 时间打折，对照 §附 测算文档稳态 980 万/家约 18.4%） |

| 版本开发费、POC / 咨询及其他技术服务 | 495 | 约 13% | 一次性或项目型 |
| **合计** | **3,800** | **100%** | |

| 项目 | 金额 | 说明 |

### 2.2 2027 年：会计净利润 2,583 万（协议下限 1,500 万）

**年度定位**：规模化扩张年；**累计合资公司 16 家**（同比仅新增 4 家，重在 2026 年 12 家存量达产 + 单家年化贡献向 §附 测算文档稳态值收敛，单家年均 ≈ 563 万），**收入扩张主引擎是 API / 平台服务（同比 +611%，至 8,100 万；增长动力见 §1.3「API 增长逻辑」——装机量消耗 + 网络裂变）**；**运营支持成本约收入的 60%**（主体为算力与可变履约，见 **§3.4**）；CapEx 进入高峰（4,000 万），由 A 轮覆盖（详见 **§四 / §五**）。

**收入结构**

| 来源 | 金额 | 占收入比 |
|------|-----:|--------:|
| 合资公司相关收入（国内放量 + 新增） | 9,000 | 50% |
| **API / token / 平台服务** | **8,100** | **45%** |
| 海外（西语系等）及其他 | 900 | 5% |
| **合计** | **18,000** | **100%** |

**支出结构**

| 项目 | 金额 | 说明 |
|------|-----:|------|

### 3.3 薪酬与全口径人力原则

「全口径人力成本」含：固定工资、绩效与项目 / 交付奖金、公司承担社保公积金、常规福利；**以及**与一线交付 / 平台运营 / 客户成功直接挂钩、按工时或项目计费的**外包劳务**（即计入「运营支持成本」科目内可识别的人力性外包支出，与"算力 / token / 云租用"等非人力部分区分）；**不含股权摊销 / 股份支付**（**2026 年期权池尚未启动，无影响；2027 年起**若审计要求按 IFRS 2 / CN GAAP 11 号将股份支付入损益，按 A 轮估值假设 5 亿、10% 期权池、4 年归属粗算约 **1,250 万 / 年**，可能影响协议利润达成——**详见 §1.4 协议口径注**）。

| 原则 | 说明 |
|------|------|
| **固浮结构** | 研发 / 管理：固定为主（约 75%–85%）+ 少量绩效；生态专员 / 一线交付：固定约 65%–75% + 与里程碑、验收、用量挂钩的浮动。 |
| **级别金字塔** | 核心研发与架构明显高于基层实施；生态专员分骨干带教、驻场交付、标准培训等带宽。 |
| **中长期激励** | 期权向研发骨干、核心管理、交付架构师倾斜；基层生态专员以现金 + 短期绩效为主。 |
| **全口径人力 / 收入** | **2026** 约 **36%–42%**；**2027** 约 **32%–38%**；**2028** 约 **28%–34%**（随平台化摊薄人均交付）。 |
| **高管与核心骨干** | 高管 + 首席 / 总监级现金成本合计 **≤ 全口径人力成本的 28%**。 |

**科目映射与五个含人力科目的边界对照（必读）**

> 警示：**「研发」≠ 全部研发投入**（仅指研发岗人力 + 研发用非工资支出，**不含**项目交付岗位）；**「商务 / BD / 共用职能」≠ 全公司团队**（仅指销售 / BD / 商事 / 共用人力，**不含**研发岗、生态专员、管理岗）；**「项目交付人力」**（生态专员 / 一线运营 / 履约外包）**全部归入「运营支持成本」**，不在"研发"或"商务 / BD / 共用职能"中。

## 四、资本性支出（CapEx）与折旧

本章解释 v1.6 起的 **CapEx 资本化** 处理：硬件、平台许可与可资本化研发投入按 **资产** 计入资产负债表，按规则分年折旧 / 摊销进损益；**§二** 各年损益表中的"资产折旧"行即由本章测算得出。

### 4.1 三年 CapEx 计划

| 年度 | 固定资产（GPU / 服务器 / 工作站） | 无形资产（许可 / 可资本化研发） | **当年 CapEx 合计** | 资金来源 |
|------|-------------------------------:|-----------------------------:|--------------------:|----------|
| 2026 | 150 | 100 | **250** | 天使轮 + 首期回款 |
| 2027 | 2,000 | 2,000 | **4,000** | A 轮 + 经营回款 |
| 2028 | 4,000 | 4,000 | **8,000** | B 轮 + 经营回款 |
| **三年合计** | **6,150** | **6,100** | **12,250** | — |

> **来源**：`未来三年资本性支出计划`（2026 / 2027 与该附件一致；2028 按提交版务实口径取 8,000 万，与附件 1.6 亿差额作为 B 轮上限弹性，**不写入本计划损益与协议测算**）。

### 4.2 折旧政策（统一）

- **方法**：直线法
- **折旧年限**：**3 年**（GPU / 服务器寿命主导，软件许可同年限同步摊销，便于统一测算与对外解释）
- **当年开始日**：当年全年计提（不做半年约定，简化口径）
- **残值**：不考虑

> 注：实务中可按"固定资产 3 年 / 无形资产 5 年"分别处理，2026–2028 折旧合计：**统一 3 年口径 5,583 万 vs 拆分口径 4,477 万，差异约 1,106 万（≈ 20%）**（v2.0 由原"约 ±10%"修正为更精确的 20%）；对协议利润均无影响——拆分口径下 2027 / 2028 净利润分别为约 **2,863 / 6,130 万**（vs 统一口径 2,583 / 5,317），仍守协议下限并留有更大安全垫。本计划取**统一 3 年**以便对外口径一目了然；后续若审计建议拆分，差额作为利润上行空间。

### 4.3 折旧分年表（直接进 §二 损益）

| 年度 | 来自 2026 CapEx 250 的折旧 | 来自 2027 CapEx 4,000 的折旧 | 来自 2028 CapEx 8,000 的折旧 | **当年合计折旧** |
|------|---------------------------:|----------------------------:|----------------------------:|---------------:|
| 2026 | 83 | — | — | **83** |
| 2027 | 83 | 1,333 | — | **1,417** |
| 2028 | 83 | 1,333 | 2,667 | **4,083** |

> **舍入说明**：分项均按 CapEx ÷ 3 精确计算后再四舍五入到整数（83.33 / 1,333.33 / 2,666.67）；**当年合计折旧**按精确数求和后再四舍五入。因此 2027 行分项相加 = 1,416，但精确加和 1,416.67 四舍五入为 1,417；2028 行分项相加 = 4,083 与精确加和 4,083.33 取整后一致。差异不超过 1 万元，对协议利润测算无影响。

### 4.4 CapEx 现金 vs 损益的关系

| 年度 | 当年 CapEx 现金支出 | 当年进损益的折旧 | 当年"现金口径减损" | 与原 v1.5 "资产/设备" 行的差异 |
|------|------------------:|---------------:|----------------:|-------------------------:|
| 2026 | 250 | 83 | 250（融资覆盖） | 原 200 → 折旧 83，**净利润 +117** |
| 2027 | 4,000 | 1,417 | 4,000（A 轮覆盖） | 原 2,600 → 折旧 1,417，**净利润 +1,183** |
| 2028 | 8,000 | 4,083 | 8,000（B 轮 + 回款） | 原 6,500 → 折旧 4,083，**净利润 +2,417** |

> **本表 delta 口径说明**：上表"净利润 +117 / +1,183 / +2,417 万"反映的是 **v1.5 → v1.6 会计处理变化**（资产/设备直接费用化 → 资本化 + 直线折旧）的影响，**不含 v1.7 因管理 / 合规科目预算上调（合计 -50 / -100 / -100 万）带来的回调**，**也不含 v2.1 因 2026 收入升级 3,500 → 3,800 带来的 +120 净利润上行**。**v2.1 当前基线净利润 477 / 2,583 / 5,317 万**见 §1.2 / §1.4 / §2.x。

**关键结论**：
1. **会计利润上行**：CapEx 资本化后，2026 / 2027 / 2028 会计净利润分别为 **477 / 2,583 / 5,317 万**（v2.1 口径，2026 由 v1.7 的 357 万升级），对协议利润下限均有显著安全垫（2026 收入 +300 万；2027: +1,083 万；2028: +2,317 万）。
2. **现金压力前置**：CapEx 现金高峰在 2027（4,000 万）与 2028（8,000 万），必须依赖 A 轮 / B 轮按节奏到位；**§五** 融资规模与之直接挂钩。
3. **若融资滞后**：可分别将 2027 / 2028 CapEx 收缩至 50–60%（推迟自有 GPU 集群、改云租用），仍可达成协议利润，但运营支持成本将上行 5–10 个百分点。

## 五、融资与执行原则

融资规模以 **下一阶段交付清单、市占目标、CapEx 节奏与现金安全垫** 为锚，与 `融资计划.md` 天使轮及 **§四** CapEx 计划衔接；**不以估值叙事抬升募资额**。

| 轮次 | 规划募资区间（人民币） | 对应 CapEx 与用途逻辑 |
|------|------------------------|-----------|
| 天使轮 | **690 万** | 已披露口径；引擎稳定化、首批商业化与组织扩编（含 2026 年 250 万 CapEx 启动部分） |
| A 轮 | **约 1.2–1.5 亿元** | 2026–2027 过渡期：覆盖 **2027 年 4,000 万 CapEx**、合资公司规模化交付、平台与 API、海外试点与合规；预留 **6–9 个月**运营缓冲（2027 月度现金支出剔除折旧后约 1,167 万元/月，6–9 个月对应 7,000–10,500 万元） |
| B 轮 | **约 2.5–4.0 亿元** | 2027–2028：覆盖 **2028 年 8,000 万 CapEx**、平台化与区域扩张、算力与基础设施、全球交付与客户成功；与经营回款协同 |

**执行原则**：单轮可在区间内随谈判与市场取值；**下轮融资若滞后**，支出按优先级收缩——**优先**已签约交付与合规，**再**按 §4.4 关键结论 3 缩减扩张性 CapEx 与市场费用。

## 六、结语（可作对外收束表述）

2026–2028 年在协议指标下可执行：**2026 年收入 3,800 万、会计净利润 477 万（协议下限 3,500 万 / +300 万安全垫），12 家合资公司目标，人员「核心 15 + 驻场 12 + 总部带教 / 中台 8」年末约 35 人，经营放量与主要成本在 5–12 月**；**2027–2028** 编制约 **105 / 174 人**，**累计合资公司从 12 家温和扩至 16 / 20 家（同比 +4 / +4，重在存量达产 + 单家年化贡献向 §附 测算文档稳态值收敛）**，**收入增速由 API / 平台主驱动**（API 同比 +611% / +233%，占收入比从约 30% 升至约 60%；增长来自已有装机量消耗 + 网络级裂变），人头增速远低于收入增速；**会计净利润 2,583 / 5,317 万**，对协议下限留有 1,083 / 2,317 万安全垫；**三年 CapEx 累计 1.225 亿元**（2026 / 2027 / 2028 = 250 / 4,000 / 8,000 万），按年度由天使（690 万） / A 轮（1.2–1.5 亿） / B 轮（2.5–4.0 亿）分级覆盖；融资与支出按市场占有与交付规模配置，并保留可执行的现金纪律。

## 附：与 `合资公司资金收入测算.md` 的口径关系

`合资公司资金收入测算.md` 基准档反映 **3 家头部合资公司在达产后** 给母公司的稳态年化收入约 **2,939 万元**（含分红 + 进化涌现归属成本收入），平均**单家稳态约 980 万元 / 年**。本计划 **§2.1** 中"合资公司股权 / 分成 / 联合商业化 = 2,165 万 / 12 家（单家年均 ≈ 180 万）"为 **2026 年首年口径**，体现以下差异：

1. **时间打折**：2026 年商业化自 5 月起量（**§3.1**），多数合资公司当年仅 4–8 个月在产，不能按全年达产推算；
2. **达产爬坡**：新增合资公司从签约到首批回款通常 6–12 个月，2026 年大多数处于爬坡期，**单家年均 180 万 / 稳态 980 万 ≈ 18.4%，符合首年爬坡区间**；
3. **按收入主体互斥拆分（避免双计，v2.0 修订口径）**：测算文档单家稳态 980 万 = 分红 + 进化涌现归属成本 / 技术服务，**两类均由合资公司主体支付**。本计划"合资公司"列 2,165 万 / 12 家**整合**了所有合资公司主体支付的款项（分红 + 股权 + 联合商业化 + 主体内部技术服务费分成）；"API / token / 平台服务费" 1,140 万**仅含合资公司主体之外的客户**（直签 API / 自助平台 / 第三方调用 / 海外）。两列**互斥不重叠**，单家 180 / 稳态 980 ≈ 18.4% 首年爬坡逻辑成立；
4. **2027 / 2028** 随合资公司达产、单家年化贡献向测算文档稳态值收敛（**单家年均 2026 ≈ 180 万 / 2027 ≈ 563 万 / 2028 = 800 万，已分别达稳态的约 18% / 57% / 82%**），"合资公司相关收入"逐年放大至 2,165 / 9,000 / 16,000 万；扩张主要靠**已签约存量爬坡 + 少量精挑新增**（家数仅 12→16→20），不依赖家数线性放大，逻辑与测算文档一致。

> 简言之：测算文档为**单家稳态**视角，本计划为**精挑组合 + 首年时间加权 + 三年向稳态收敛**视角，两者口径不同但不矛盾；2028 起两口径在单家量级上已显著趋同（800 万 vs 980 万）。
