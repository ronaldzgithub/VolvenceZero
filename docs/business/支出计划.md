# Volvence 2026–2028 年度支出计划（提交版）

**文档性质**：对外 / investor 提交用财务与支出规划。  
**版本**：提交版 v1.6  
**货币**：人民币（万元）  
**阅读结构**：**总览**（目标与三年一盘棋）→ **分年**（2026 / 2027 / 2028 收入、支出、损益与里程碑）→ **人力与科目口径**（时间节奏、编制、薪酬、与报表勾稽）→ **资本性支出与折旧** → **协议与融资**。

**v1.6 修订要点**：(1) 修正 2026 支出加总，净利润口径调整；(2) 资产/设备改按 **资本化 + 直线折旧** 入损益（详见 **§四**），原"资产/设备"行改为"资产折旧"；(3) 三年 CapEx 计划与折旧分年表独立成章。

## 一、总览：要达成什么、三年长什么样

### 1.1 与协议及战略主线对齐

本计划对齐增资协议及补充协议核心经营指标：**2026 年收入不低于 3,500 万元；2027 年净利润不低于 1,500 万元；2028 年净利润不低于 3,000 万元。**  

商业结构上强调三件事：**API / 平台类收入占收入比逐年明显提高**；**2026 年完成 20 家合资公司落地**并形成可复制交付；**融资各轮按「占有市场与规模化交付」的真实资源需求定规模**，保留安全垫，不为叙事抬升募资额。

### 1.2 三年核心数据一览

| 年度 | 实际收入 | 总支出（含折旧） | **会计净利润** | 当年新增 CapEx | 协议 / 规划硬约束 | API / 平台类占收入比 |
|------|--------:|---------------:|---------------:|---------------:|-------------------|---------------------:|
| 2026 | **3,500** | **3,093** | **407** | **250** | 收入 ≥ 3,500 万；**20 家合资公司** | **约 30%** |
| 2027 | **18,000** | **15,317** | **2,683** | **4,000** | 净利润 ≥ 1,500 万 | **约 45%** |
| 2028 | **45,000** | **39,583** | **5,417** | **8,000** | 净利润 ≥ 3,000 万 | **约 60%** |

> **口径说明**：表中"总支出"含资产折旧（按 3 年直线计提，详见 **§四**），不含 CapEx 现金支出本身；"会计净利润"为会计口径，对应协议利润约束。**当年新增 CapEx** 为现金口径的资本性支出，单独通过 A/B 轮融资与经营回款支持，不重复进损益（详见 **§四**）。

**API / 平台类收入**（全文同口径）：含按调用量计费的 API、token / 平台服务费，及与平台化交付直接相关的技术服务；与股权分成、一次性项目费区分列示，便于逐年比较占比。

### 1.3 执行节奏与组织规模（提要）

- **2026 年**：商业化与大规模交付自 **5 月**起量，**人员扩张、算力与履约相关支出主要集中在 5–12 月（8 个月）**；损益仍按 **12 个月完整年度**列示。年末编制 **约 35 人**：**总部核心 15 人（研发 8 + 运营 4 + 管理 3）** + **20 家合资公司驻场教辅 20 人**；全年 **加权人年约 20–24**。  
- **2027–2028 年**：完整经营年度，年末编制分别约 **105 人、174 人**（职能口径见 **第三章**）；**收入增速主要由 API / 平台服务驱动**（API 占比 30% → 45% → 60%），合资公司家数 **2027 年累计 30 家、2028 年累计 35 家**——增速明显放缓，重在已签约存量达产，**人头增速远低于收入增速**。

**三年合资公司家数与 API / 平台收入对照**

| 年度 | 累计合资公司（家） | 同比新增 | 合资公司相关收入 | API / 平台服务收入 | API 同比 |
|------|-----------------:|--------:|----------------:|------------------:|--------:|
| 2026 | 20 | +20 | 2,000 | 1,050 | — |
| 2027 | **30** | +10 | 9,000 | **8,100** | **+671%** |
| 2028 | **35** | +5 | 16,000 | **27,000** | **+233%** |

> 2027 / 2028 收入扩张的**主引擎是 API / 平台**（同比 +671% / +233%），合资公司端转向**已签约存量的达产爬坡 + 少量精挑新增**；不依赖合资公司家数线性放大。

### 1.4 与协议条款逐条对照（总表）

| 条款 | 要求 | 本计划（会计口径） | 安全垫 |
|------|------|--------|--------|
| 补充协议收入 | 2026 年收入 ≥ 3,500 万 | **3,500 万** | 持平达标 |
| 补充协议利润 | 2027 年净利润 ≥ 1,500 万 | **2,683 万** | +1,183 万 |
| 补充协议利润 | 2028 年净利润 ≥ 3,000 万 | **5,417 万** | +2,417 万 |

> 协议利润口径为**会计净利润**（资产折旧入损益、CapEx 资本化）。本计划自 v1.6 起统一按该口径列示，对协议指标均留有安全垫；详见 **§四 资本性支出与折旧**。

## 三、组织、人力与薪酬：横切说明（细节）

本章回答：**人何时进场、多少人、钱怎么定、报表科目怎么拼**，与 **第二章** 分年数字交叉印证。

### 3.1 第一年成本何以集中在 5–12 月

2026 年商业化放量自 **5 月**启动，**人员扩张、算力与履约相关支出主要发生在 5–12 月共 8 个月**；1–4 月侧重引擎稳定、合资合同闭环、交付预备及小规模刚性支出。**收入与净利润仍按完整自然年度列示**；现金流与滚动预算与「**5 月起进入主要经营期**」对齐。

### 3.2 职能口径与编制预测（年末在编，单位：人）

**研发**：算法、工程、产品技术。**运营**：平台运维、SRE、内容安全与审核、客户支持底盘、商务运营与项目管理。**管理**：高管、财务、法务、人力资源、行政。**教辅**：客户培训、交付赋能、文档与现场辅导、客户成功及履约驻场 / 集训——**2026 年按 20 家 × 各 1 名驻场计 20 人**。

| 年度 | 研发 | 运营 | 管理 | 教辅 | **合计** |
|------|-----:|-----:|-----:|-----:|--------:|
| 2026（末） | 8 | 4 | 3 | 20 | **35** |
| 2027（末） | 26 | 20 | 9 | 50 | **105** |
| 2028（末） | 48 | 34 | 14 | 78 | **174** |

**2026 编制模型**：总部 **15 人** + **驻场 20 人**（20 家 × 各 1 名），不采用总部堆人替代一线；驻场以 **标准化交付、培训与客户成功** 为主，复杂方案由总部与外包兜底。

**2027 年（30 家合资）**：教辅 **50 人**对应 **30 家** + 区域中台与客户成功——平均 **约 1.67 人 / 家**，明确**区域共享 + 总部教辅 + 中台**，**不与**合资主体家数机械「一司一人」。**2027 年新增合资公司仅 10 家**，组织扩张主要服务于 **2026 年 20 家存量达产** 与 API / 平台业务的客户成功。

**2028 年（35 家合资）**：教辅 **78 人**对应 **35 家** + 平台支持面扩大——平均 **约 2.23 人 / 家**，含驻场 + 区域中台 + 客户成功 + 平台运营。**2028 年合资公司新增仅 5 家**，**人头大头转向 API / 平台业务的客户成功与全球化运营**，与 §1.3 "API 主导收入扩张" 一致；**不重复计算**纯算力 / 云外包与编制人头。

### 3.3 薪酬与全口径人力原则

「全口径人力成本」含：固定工资、绩效与项目 / 交付奖金、公司承担社保公积金、常规福利；**不含**股权摊销（单立激励方案）。

| 原则 | 说明 |
|------|------|
| **固浮结构** | 研发 / 管理：固定为主（约 75%–85%）+ 少量绩效；教辅 / 一线交付：固定约 65%–75% + 与里程碑、验收、用量挂钩的浮动。 |
| **级别金字塔** | 核心研发与架构明显高于基层实施；教辅分骨干带教、驻场交付、标准培训等带宽。 |
| **中长期激励** | 期权向研发骨干、核心管理、交付架构师倾斜；基层教辅以现金 + 短期绩效为主。 |
| **全口径人力 / 收入** | **2026** 约 **36%–42%**；**2027** 约 **32%–38%**；**2028** 约 **28%–34%**（随平台化摊薄人均交付）。 |
| **高管与核心骨干** | 高管 + 首席 / 总监级现金成本合计 **≤ 全口径人力成本的 28%**。 |

**人均全口径成本参考（万元 / 人 / 年，测算锚点）**

| 年度 | 研发 | 运营 | 管理 | 教辅 |
|------|------|------|------|------|
| 2026 | 42–68 | 24–40 | 38–72 | 22–38 |
| 2027 | 40–72 | 26–44 | 36–75 | 24–42 |
| 2028 | 38–75 | 28–48 | 35–78 | 25–45 |

**科目映射（摘要）**：教辅与一线运营人工 **主要**计入「运营支持成本」；研发人力 **主要**计入「研发」；管理人力计入「管理 / 合规」或按规定分摊；**以财务制度为准**。

### 3.4 人力数字与利润表科目的勾稽（必读细节）

2027–2028 年「运营支持成本」约为收入的 **60%**，其中 **主体**是随用量变动的 **算力、token、云与外包交付**；教辅与一线运营人工仅为其中一部分，**不能**将本科目总额等同于工资总额。

**四类编制表**不单独列市场 / BD 人头（避免与「运营」重复统计）；BD 与商事人力及外包在 **「团队」**与 **「市场」**。比较 **§3.3** 中「全口径人力 / 收入」目标时，须在 **同一口径**下纳入上述人员及项目外包、驻场补充等履约劳务；**不含**股权摊销。「研发」「管理 / 合规」等行另含云工具、审计、外聘律师等 **非工资** 支出，故 **不能**用「表上人 × §3.3 人均」轧平整张表。

按人均区间 **中位**粗算，仅四类全职工部分约 **2027 年 0.43 亿元、2028 年 0.75 亿元**；叠加「团队」「市场」中的 BD 与外包后，可与 **§3.3** 人力 / 收入目标带衔接。

## 四、资本性支出（CapEx）与折旧

本章解释 v1.6 起的 **CapEx 资本化** 处理：硬件、平台许可与可资本化研发投入按 **资产** 计入资产负债表，按规则分年折旧 / 摊销进损益；**§二** 各年损益表中的"资产折旧"行即由本章测算得出。

### 4.1 三年 CapEx 计划

| 年度 | 固定资产（GPU / 服务器 / 工作站） | 无形资产（许可 / 可资本化研发） | **当年 CapEx 合计** | 资金来源 |
|------|-------------------------------:|-----------------------------:|--------------------:|----------|
| 2026 | 150 | 100 | **250** | 天使轮 + 首期回款 |
| 2027 | 2,000 | 2,000 | **4,000** | A 轮 + 经营回款 |
| 2028 | 4,000 | 4,000 | **8,000** | B 轮 + 经营回款 |
| **三年合计** | **6,150** | **6,100** | **12,250** | — |

> **来源**：`未来三年资本性支出计划`（2026 / 2027 与该附件一致；2028 按提交版务实口径取 8,000 万，与附件 1.6 亿差额作为 B 轮上限弹性，**不写入本计划损益与协议测算**）。

### 4.2 折旧政策（统一）

- **方法**：直线法
- **折旧年限**：**3 年**（GPU / 服务器寿命主导，软件许可同年限同步摊销，便于统一测算与对外解释）
- **当年开始日**：当年全年计提（不做半年约定，简化口径）
- **残值**：不考虑

> 注：实务中可按"固定资产 3 年 / 无形资产 5 年"分别处理，两种口径下 2026–2028 折旧合计差异约 ±10%，对协议利润均无影响；本计划取 **统一 3 年** 以便对外口径一目了然。后续若审计建议拆分，差额作为利润上行空间。

### 4.3 折旧分年表（直接进 §二 损益）

| 年度 | 来自 2026 CapEx 250 的折旧 | 来自 2027 CapEx 4,000 的折旧 | 来自 2028 CapEx 8,000 的折旧 | **当年合计折旧** |
|------|---------------------------:|----------------------------:|----------------------------:|---------------:|
| 2026 | 83 | — | — | **83** |
| 2027 | 83 | 1,333 | — | **1,417** |
| 2028 | 83 | 1,333 | 2,667 | **4,083** |

### 4.4 CapEx 现金 vs 损益的关系

| 年度 | 当年 CapEx 现金支出 | 当年进损益的折旧 | 当年"现金口径减损" | 与原 v1.5 "资产/设备" 行的差异 |
|------|------------------:|---------------:|----------------:|-------------------------:|
| 2026 | 250 | 83 | 250（融资覆盖） | 原 200 → 折旧 83，**净利润 +117** |
| 2027 | 4,000 | 1,417 | 4,000（A 轮覆盖） | 原 2,600 → 折旧 1,417，**净利润 +1,183** |
| 2028 | 8,000 | 4,083 | 8,000（B 轮 + 回款） | 原 6,500 → 折旧 4,083，**净利润 +2,417** |

**关键结论**：
1. **会计利润上行**：CapEx 资本化后，2026 / 2027 / 2028 会计净利润分别为 **407 / 2,683 / 5,417 万**，对协议利润下限均有显著安全垫。
2. **现金压力前置**：CapEx 现金高峰在 2027（4,000 万）与 2028（8,000 万），必须依赖 A 轮 / B 轮按节奏到位；**§五** 融资规模与之直接挂钩。
3. **若融资滞后**：可分别将 2027 / 2028 CapEx 收缩至 50–60%（推迟自有 GPU 集群、改云租用），仍可达成协议利润，但运营支持成本将上行 5–10 个百分点。

## 五、融资与执行原则

融资规模以 **下一阶段交付清单、市占目标、CapEx 节奏与现金安全垫** 为锚，与 `融资计划.md` 天使轮及 **§四** CapEx 计划衔接；**不以估值叙事抬升募资额**。

| 轮次 | 规划募资区间（人民币） | 对应 CapEx 与用途逻辑 |
|------|------------------------|-----------|
| 天使轮 | **690 万** | 已披露口径；引擎稳定化、首批商业化与组织扩编（含 2026 年 250 万 CapEx 启动部分） |
| A 轮 | **约 1.0–1.5 亿元** | 2026–2027 过渡期：覆盖 **2027 年 4,000 万 CapEx**、合资公司规模化交付、平台与 API、海外试点与合规；预留 **6–9 个月**运营缓冲 |
| B 轮 | **约 2.5–4.0 亿元** | 2027–2028：覆盖 **2028 年 8,000 万 CapEx**、平台化与区域扩张、算力与基础设施、全球交付与客户成功；与经营回款协同 |

**执行原则**：单轮可在区间内随谈判与市场取值；**下轮融资若滞后**，支出按优先级收缩——**优先**已签约交付与合规，**再**按 §4.4 关键结论 3 缩减扩张性 CapEx 与市场费用。

---

## 六、结语（可作对外收束表述）

2026–2028 年在协议指标下可执行：**2026 年收入 3,500 万、会计净利润 407 万，20 家合资公司目标，人员「核心 15 + 驻场 20」年末约 35 人，经营放量与主要成本在 5–12 月**；**2027–2028** 编制约 **105 / 174 人**，**累计合资公司从 20 家温和扩至 30 / 35 家（同比仅 +10 / +5，重在存量达产）**，**收入增速由 API / 平台主驱动**（API 同比 +671% / +233%，占收入比从约 30% 升至约 60%），人头增速远低于收入增速；**会计净利润 2,683 / 5,417 万**，对协议下限留有 1,183 / 2,417 万安全垫；**三年 CapEx 累计 1.225 亿元**（2026 / 2027 / 2028 = 250 / 4,000 / 8,000 万），按年度由天使 / A / B 轮分级覆盖；融资与支出按市场占有与交付规模配置，并保留可执行的现金纪律。

## 附：与 `合资公司资金收入测算.md` 的口径关系

`合资公司资金收入测算.md` 基准档反映 **3 家头部合资公司在达产后** 给母公司的稳态年化收入约 **2,939 万元**（含分红 + 进化涌现归属成本收入）。本计划 **§2.1** 中"合资公司股权 / 分成 / 联合商业化 = 2,000 万 / 20 家"为 **2026 年首年口径**，体现以下差异：

1. **时间打折**：2026 年商业化自 5 月起量（**§3.1**），多数合资公司当年仅 4–8 个月在产，不能按全年达产推算；
2. **达产爬坡**：新增合资公司从签约到首批回款通常 6–12 个月，2026 年大多数处于爬坡期；
3. **仅含分红与权益类**：2026 年口径里"合资公司"列只含分红 / 股权 / 联合商业化，**进化涌现归属的技术服务 / API 部分已计入"API / token / 平台服务费 1,050 万"**——避免双计；
4. **2027 / 2028** 随合资公司达产、单家年化贡献接近测算文档稳态值，"合资公司相关收入"逐年放大至 9,000 / 16,000 万，逻辑与测算文档一致。

> 简言之：测算文档为**单家稳态**视角，本计划为**组合 + 首年时间加权**视角，两者口径不同但不矛盾；2027 起两口径在数量级上趋同。
