## 第一部分 — Xfund 投资理念深度研究（事实底稿）

> 本节是后续 PPT 设计的"事实地基"。每一条都对应公开可查的来源（Patrick 自己说过、xfund 官方文章、可被 due diligence 团队复核的事实），不是营销话术。

### 1.1 基金结构与历史

| 维度 | 事实 | 含义 |
|---|---|---|
| **成立年份** | 2012 年（原名 *The Experiment Fund*） | "Experiment" 这个词本身是品牌资产——他们承认自己投的是"实验"，不是 thesis-driven |
| **三只基金规模** | $10M（2012）→ $100M（2014, Xfund 2）→ $120M（2020, Xfund 3，oversubscribed） | 规模刻意保持中小——这是**精品店模型**，不是平台型 VC |
| **LP 阵容** | Harvard University + NEA + Breyer Capital + Accel + Polaris | LP 本身就是顶级机构 + 顶级大学的耦合——Patrick 的决策有 institutional weight |
| **办公室** | Cambridge（Harvard 边）+ Palo Alto（Stanford 边） | 双校园战略——故意贴住两个顶级学府的 talent flow |
| **Xfund 2 表现** | 已 return 1/3 of invested capital（截至 2020 公开口径） | "top tier of VC performance benchmarks"——业绩证明这套打法 work |
| **Xperiment Stake**（在跑） | $100K SAFE，针对 Harvard / Yale / MIT / UPenn / Stanford / Berkeley 在校学生 | 把"first investor"地位**制度化**——让他们的"90% 是 first money"持续兑现 |

### 1.2 投资阶段与定位

- **Stage**: pre-seed / seed / Series A
- **Sector**: 全 sector，但偏 AI / fintech / healthcare / consumer tech
- **Check size**: 第一张支票通常 $250K–$3M（依情况定，Xperiment Stake 起步 $100K）
- **签约角色**: **90% 案子是 first money**——这是 xfund 最显著的标签
- **决策模式**: 由 partner 直接拍板（fund 小，不需要 IC 多轮）；依赖 faculty referral 和 Patrick 个人直觉，不依赖严格 quant screening

### 1.3 Patrick Chung 个人画像（最重要部分）

> 投 xfund 本质上是投 Patrick 个人的判断品味。他不是 thesis-driven VC，他是**人格 + 直觉 + 关系**驱动的 VC。

| 维度 | 事实 | 对 deck 设计的含义 |
|---|---|---|
| **教育** | Oxford University College（文科背景）+ Harvard | 他自己就是 liberal-arts background，他偏爱看到自己的影子 |
| **职业起点** | 早期在 NEA | 2005 年作为 NEA 投资人写下 19 岁 Sam Altman / Loopt 的第一张机构支票（$5M） |
| **创立 Xfund** | 2012 年与 Brandon Farwell 联合创立 | 把"投 first-time、跨学科、非典型 founder"做成一个独立基金 |
| **被广泛引用的一句话** | *"We chase the talent. We let the talent decide where to go."* | 这是他的**人格底色**——不卡 thesis、不卡 sector，认人 |
| **被广泛引用的另一句** | *"You can be a poet and a physicist. A hacker and a historian."* | 他偏爱**多维洞察的复合品味**，反感"只是个工程师"的窄人格 |
| **从 Sam Altman 那场投资学到的三条**（自己说的） | 1) 早期阶段最重要的是 incredible team；2) CEO 很孤独，好的董事必须是 steadfast support；3) **关系**是比公司本身更持久、更有意义的东西 | 第三条直接呼应我们卖的"关系机器"——这是 deck 必须打的钩 |

### 1.4 Xfund 投资理念的四条底层公理

把官方话术、Patrick 自己的访谈、portfolio 行为 cross-reference 后提炼出的**四条底层公理**：

#### 公理 A: founder archetype 必须被扩张

> 原话：*"Most venture firms today will only talk to people who majored in computer science or engineering. Xfund will, of course, invest in computer scientists and engineers — but we also seek out history majors, English majors, economics majors, and classics majors."*

含义：传统 VC 把 founder 限定为 CS/工程背景，xfund 把这个 archetype 撕开。这不是政治正确，是**生意角度的判断**——他们认为复杂问题需要跨学科洞察才能解。

证据：
- Daniel Nadler（Kensho 创始人）= Harvard PhD 经济学家 + 诗人 + 导演
- Anthemos Georgiades（Zumper 创始人）= Oxford classics 专业（古典学，拉丁文 + 希腊文）
- Anne Wojcicki（23andMe）= Yale 生物 + Stanford 经济
- Tuan Ho（Philo 创始人）= Harvard pre-med（差点去医学院）

#### 公理 B: vertical proprietary data 是 startup 在 AI 时代唯一的护城河

> 原话（2024-07 访谈）：*"No one is going to compete with the established LLMs. They're simply too expensive to build and operate. But start-ups can licence proprietary data to develop better trained LLMs."*

含义：他**已经放弃**让 startup 在 LLM scaling 上赢大厂的幻想。他相信的是 vertical data + 行业 hold。

证据：
- **Open Evidence**（医疗 AI）— 拿到 Mayo Clinic 数据
- **Delphi**（数字克隆）— 用个人 messages + 工作决策训练个人化 AI
- **Kensho**（金融 AI，已 acquire by S&P Global）— "largest AI deal in history"，靠的也是 financial domain data

这意味着对 VolvenceZero pitch xfund 时，**"我们不与大厂竞争 LLM scaling"必须放在第一页**——这正好是 Patrick 自己的判断。

#### 公理 C: 90% 案子是 first money — "tipping into existence"

> 原话：*"In 90% of all cases, Xfund is the very first investor in our companies. More often than not, we have 'tipped' our founders into existence."*

含义：xfund 不是"加入 round"的 VC，他们是**催化 founder 决定 quit default path** 的 VC。

证据（每个都是"原本要走 default path 但被 xfund 救出来"）：
- Sam Altman / Loopt — 19 岁 Stanford 二年级辍学
- Tuan Ho / Philo — 离开医学院
- Nick Krasney / Philo — 离开法学院
- Daniel Nadler / Kensho — 离开学术
- Anthemos Georgiades / Zumper — 离开 BCG management consulting

对 deck 的含义：**Patrick 看的不是"项目能不能投"，是"这个人现在是不是处于他人生重要的转折点"**。我们的 pitch 必须给他这种"if not now, this person will go back to default path" 的紧迫感。

#### 公理 D: 大学是发现 talent 的引水管，但需要三个文化前提

> 原话（2024-07 访谈）：大学要支持创业商业化需要三件事——
> 1) Cultural shift（商业化要被认可）
> 2) Sophisticated IP licensing
> 3) Flexibility（学者出入企业自由）

对 deck 的含义：他对"学术 + 工业 + 投资"的三角关系有**结构性思考**。我们如果能展示出"我们的研究密度像学术、纪律像工业、商业化路径像 VC"，他会认。

### 1.5 Xfund Portfolio 的 5 个分类（按内在逻辑而非 sector）

把 portfolio 按"为什么 xfund 投了"重新分类，会比按 sector 分类更有信息：

| 类别 | 代表案例 | 共性 |
|---|---|---|
| **A. 改写 founder 默认路径的非典型创始人** | Loopt / Philo / Kensho / Zumper / Patreon | 创始人本身有"非 CS 背景 + 多维洞察"的人格 |
| **B. 专有数据 vertical AI** | Open Evidence (Mayo) / Kensho (S&P) / 23andMe (基因) | 通过独家 corpus 形成 unbeatable 数据壁垒 |
| **C. 关系/人格的产品化** | Delphi (digital clones) / Patreon (creator economy) / Replika 类邻居 | 把"人本身"作为产品的核心载体 |
| **D. 改写传统行业的工具** | Plaid (金融基础设施) / Robinhood (零售券商) / Gusto (薪酬) / Zumper (租房) | LP 看得懂 + 大市场 + 工程纪律 |
| **E. 教育/社交/地理位置垂直** | Andela (非洲工程师) / BoldVoice (口音训练) / AeroVect (机场自驾) | 切口很小但可扩张 |

VolvenceZero 落在 **A + B + C 三类的交集**——"非典型 founder × vertical proprietary data × 关系产品化"。**这是 portfolio 中没有任何单个公司同时占据的位置**。这是 deck 必须明确指出的"position arbitrage"。

### 1.6 Patrick Chung 个人会被什么打动 / 不会被什么打动

> 这是 deck 设计最关键的输入。投人不投 thesis 的 VC，意味着 deck 不能用 "TAM × growth × moat × ask" 这种麦肯锡式套路。

**会被打动的（按权重排序）**：
1. **founder 的人格层次感**——是否有"诗人 + 工程师"那种复合气质
2. **思想原创性**——是否有自己消化过、不可复制的判断框架
3. **关系叙事**——是否能把"为什么我必须做这件事"讲成有人物弧线的故事
4. **工程纪律的硬证据**——是否有 ship 的东西、有可被 due diligence 复核的实物
5. **anti-claim 的成熟度**——是否能主动说出"我不在做什么 / 我做不到什么"
6. **与 portfolio 的对话感**——是否能把自己定位在他已投公司的网络里（不是竞争而是延伸）

**不会被打动的（明确 turn-off）**：
1. **TAM = $XB by 2030 这种麦肯锡式数字**——他自己就是 Harvard MBA，对这套套路免疫
2. **一上来就讲 product roadmap 和功能**——他先认人再认产品
3. **"我们要 disrupt OpenAI"这种过分的口号**——他自己就是 Sam Altman 的第一投资人，知道 OpenAI 的厚度
4. **"We are pre-revenue but…"** 类似的辩解——他投 first money，不期待 revenue；但不希望听到带歉意的语气
5. **过度强调"我们是 AI 公司"**——他不投 sector，他投人

## 第二部分 — PPT 设计原则（基于研究的反推）

> 普通 VC pitch deck 的 25-page 套路（Sequoia template）对 xfund 是错配。下面 8 条原则是基于第一部分研究**反推**出来的。

### 原则 1: 第一页是人，不是产品

xfund 投人。第一页投影应该是**创始人面孔 + 一句人格化判断**，不是 logo + tagline。**让 Patrick 在 30 秒内"认识这个人"**，而不是"理解这个项目"。

### 原则 2: 用他自己的话开门，不要用我们的话

deck 的第 2-3 页必须**直接引用 Patrick 自己说过的话**，然后回答"你说过 X，我们正好是 X 的活体证据"。这是认知对齐的最快路径。

### 原则 3: 哲学叙事 + 工程证据双轨呈现

Patrick 是 Oxford 文科 + Harvard MBA + NEA 工程化 VC 的复合背景。他既能听哲学叙事（认 framework），也能识破工程造假（认 evidence）。Deck 必须**每个哲学论点配一组工程证据**，反之亦然。

### 原则 4: 必须主动占位 portfolio 网络

不能让 Patrick 在脑里去"摆放"我们与 Open Evidence / Delphi / Kensho 的关系。我们必须**自己把位置画好**给他看——"我们是你 thesis 第三块拼图"是必须落地的钩子。

### 原则 5: anti-claims 必须早出现

不要把"我们不卖什么"放在 deck 末尾。**应该在中段就主动列出**——这是给 liberal-arts VC 的成熟度信号。Patrick 看团队是否成熟看 anti-claims 的诚实度。

### 原则 6: 数字必须可被 due diligence 复核

每个数字（96 contract test / 1063 zero regression / 28 PASS benchmark / unit economics）必须**对应到仓库 main 分支某个文件 / 某次 CI run**。Deck 上不写"我们有 1000+ 测试"，要写"`tests/contracts/` 1063 个，可现场抽样跑"。

### 原则 7: 最小可视化、最大留白

Patrick 是 Harvard liberal arts 出身——他对**信息密度的审美高于硅谷常规**。deck 必须避免任何"4 quadrant matrix"、"feature comparison table 长成 8 列"、"funnel diagram with 6 stages"这种麦肯锡视觉。每页**一个论点 + 一组证据 + 一句记忆点**。

### 原则 8: 把"为什么是现在"埋在每一页

Patrick 投 Sam Altman 那次，他后来反复说的就是"timing"。我们的 deck 要让他在每一页都能感觉到"如果现在不投，这件事会被 OpenAI 在 12 个月内吃掉 / 或者这个团队会去做别的事"——**紧迫感不是靠"我们要快"喊出来的，是靠每一页 evidence 自然渗出**。

## 第三部分 — Slide-by-Slide 完整蓝图（27 页 deck）

> 格式约定：
> - **Slide #** = 页码
> - **Layout** = 视觉布局意图（设计师直接执行）
> - **On-screen** = 投影上的实际文字（≤30 字一页主标题，主体内容控制）
> - **Speaker note** = 讲者口语化补充（≤45 秒一页）
> - **Why this slide** = 这页存在的设计目的

### 总体节奏

```
Section 1 (P1-P3)  — 钩子：3 分钟
Section 2 (P4-P7)  — 业界图景与 timing：5 分钟
Section 3 (P8-P12) — 我们造了什么 + 谁在造：8 分钟
Section 4 (P13-P15)— IQ + EQ 怎么涌现：4 分钟
Section 5 (P16-P18)— 与 xfund 已投 portfolio 的对话：3 分钟
Section 6 (P19-P22)— 12-18 个月正在做的三件事：4 分钟
Section 7 (P23-P24)— 我们不卖什么 + 风险地图：2 分钟
Section 8 (P25-P27)— Ask + Close：1 分钟
                                  ─────
                              共 30 分钟
```

### Section 1 — 钩子（P1–P3）

#### Slide 1 — 封面

**Layout**: 全屏深色背景（#0A0E27 或深墨绿）。中央**不放 logo**，放一行字。

**On-screen**:
> **A bounded, continuously adapting digital organism.**
>
> **VolvenceZero**　|　Cambridge × Hangzhou
> for Patrick Chung, Xfund　|　2026

**Speaker note**:
> "Patrick，谢谢你抽出这 30 分钟。我们今天不打算给你看一个 chatbot demo——这个赛道你比我们看得多。我们想给你看一个团队花了一年时间，把 cognitive AI 的前沿地图扫干净后得出的一个判断：**有一块设计空间所有人都没碰，那就是养成式的、被工程契约约束的、活的关系机器**。"

**Why this slide**: 不用 logo + tagline 的硅谷套路；用一句"organism"替代"product"——直接打 Patrick 喜欢的那种**非典型创始人语言密度**。

---

#### Slide 2 — 一张面孔 + 一组身份

**Layout**: 左 2/5 创始人面孔（黑白纵向人像，避免微笑商务照），右 3/5 五行字（每行 8-12 字，行距大）。

**On-screen**（右侧五行）:
> **[创始人姓名]**
>
> 工程师，前 [关键履历 1]
> 文科训练背景：[读过的书 / 写过的东西 / 跨学科锚点]
> 一年前停掉所有外部工作，全职做 VolvenceZero
> 我做这件事是因为——[一句人格化原因]

**Speaker note**:
> "在讲项目前，先讲我自己。我不是典型的'AI 创业者'。我[简述跨学科背景]——这件事让我看东西的角度跟硅谷主流不太一样。我做 VolvenceZero 不是为了追 LLM 红利，是因为我相信**关系连续性是 AI 时代真正会被市场反复定价的稀缺资源**——不是更聪明的对话，是被记住的对话。"

**Why this slide**: 直接呼应 xfund 公理 A（扩张 founder archetype）和原则 1（第一页是人）。Patrick 看到"工程 + 文科 + 一年 quit default path"会立即对位他的 portfolio 模式（Daniel Nadler / Anthemos / Tuan Ho）。

> ⚠️ **待 BOSS 确认**：这一页是 deck 真正的人格化锚点，需要 BOSS 自己填三件事——
> 1. 关键履历（前公司 / 学历 / 技术贡献）
> 2. 跨学科训练（文科背景或多维兴趣的具体载体）
> 3. "为什么必须做这件事"的一句话原因
>
> 这一页填不出来或填得不真诚，整个 deck 的 conversion rate 会显著下降——它是 xfund 这种 VC 的**主要决策输入**。

#### Slide 3 — 为什么是 Xfund

**Layout**: 三段对照式，每段一行 quote + 一行回应。文字偏密但留白充足。

**On-screen**:
> **Why we're talking to you, and not anyone else.**
>
> ──
>
> 你说过：*"We chase the talent. We let the talent decide where to go."*
> 我们正好是这种 talent——liberal-arts 训练 × 工程交付 × 不在主流 thesis 上。
>
> 你说过：*"No one will compete with established LLMs. Startups can licence proprietary data."*
> 我们正好做的就是这件事的下一层——不只 vertical data，还有 vertical **runtime**。
>
> 你说过：*"It's the relationships we form that are the most meaningful, persistent, and valuable."*
> 我们造的就是这件事本身的工程载体——一台记得关系、修复关系、被监管观察的关系机器。

**Speaker note**:
> "这一页我没什么要讲的，你三句话已经讲清楚了我们为什么应该今天坐在这里。我接下来的 28 分钟就是把'我们正好是 X' 的三个'X'，每个都给你一组工程证据。"

**Why this slide**: 这是整个 deck **最重要的一页**。投资人最听不进去的是别人解释自己——**最听得进去的是被自己的话打动**。三段引用全部是 Patrick 公开说过的，他会立即认出来，并且立即解除 pitch 的防御姿态。

### Section 2 — 业界图景与 timing（P4–P7）

#### Slide 4 — Cognitive AI 的多极地图（2026）

**Layout**: 一张星图——左上 OpenAI（IQ scaling 工程整合），左下 Anthropic（alignment science），右上 DeepMind（世界模型 + 自改进），右下 Sutskever / SSI（沉默），中央留一块**空白圈**写 "?"。

**On-screen**（图下两行）:
> 我们花了 12 个月把 cognitive AI 前沿扫干净。结论：**赛道分裂成多极，互相不重叠，中间有一块"养成式数字生命"没人做**。
>
> *Source: 2024-2026 三份独立调研，OpenAI 12 人 + DeepMind 13 人 + NL/ETA 学术源头共 100+ 篇精读。*

**Speaker note**:
> "不靠堆论文摘要——一句话：OpenAI 进入工程整合期没有 paradigm 突破；Anthropic 在 alignment 实证上反超；DeepMind 走世界模型 + AlphaEvolve 走得最远；Sutskever 选择刻意沉默；Karpathy 退出前沿去做教育。整个赛道**没有任何一家在做**'养成式 + 多时间尺度 + 关系连续 + 治理可证'。这就是我们的位置。"

**Why this slide**: 给 Patrick 一个**他可能没有的 frame**——他知道每家在做什么，但**未必把"中间空白"画过出来**。我们用一个图就把空白圈出来。

#### Slide 5 — 你的判断我们已经在执行

**Layout**: 左侧大字一句 Patrick 的引用，右侧小字三行我们的对应行动。

**On-screen**:
> **"No one is going to compete with established LLMs."**
> *— Patrick Chung, 2024-07*
>
> ──
>
> 我们不训练自己的基础大模型。
> 我们用 OpenAI / Anthropic / Qwen / DeepSeek 最好的开源 + 闭源模型作**冻结底盘**。
> GPT-5 出 / Claude 5 出 / Qwen4 出——我们的关系机器自动变得更聪明。
>

## 第六部分 — 设计与制作清单（给设计师 / 演讲者）

### 视觉设计要求

- **色调**：深墨绿 / 深蓝 / 黑底白字为主，避免任何 saturated 颜色（蓝紫渐变这种 AI startup 套路要避免）
- **字体**：英文用 Inter / 中文用思源宋体（serif） 或思源黑体——**不要用 Arial / Helvetica / 微软雅黑这种默认字体**
- **图标**：用 Lucide / Phosphor 这种 minimalist line icon，**禁用任何 3D / glossy / colorful 图标**
- **图表**：禁用 PowerPoint 内置图表样式；用 D3 风格手绘图或 Apple Keynote 默认风格
- **留白**：每页**至少 30% 留白**——Patrick 的审美阈值高于硅谷常规

### 演讲准备

- **语速**：每分钟 200-220 字（中文）/ 130-150 词（英文）
- **停顿**：P3 / P14 / P18 / P26 后必须停 3-5 秒，让信息落地
- **眼神**：Patrick 投人不投 thesis——直视他超过总时长的 60%
- **手势**：禁用任何"想象一下"、"假设"这种推销手势——他会立即识别为话术

### 可能的问题准备（Q&A 必答清单）

| 问题 | 答题骨架 |
|---|---|
| "你们 burn rate 多少？" | 直接给数字 / runway / next milestone 触发的成本拐点 |
| "为什么不直接给 OpenAI / Anthropic 做 partner？" | 他们的 incentive 与我们的 vertical 不一致；我们需要的是基底，不是 GTM 共谋 |
| "团队会不会被大厂挖走？" | retention 设计 + vesting + 已经 ship 的实物本身是 retention 引力 |
| "中国市场 vs 全球市场怎么取舍？" | 短期：中国市场私域顾问跑通；中期：figure / B2B 必然全球；treat as portfolio not exclusive |
| "你说不是 AGI，但 cognitive AGI 容器最终值多少？" | 不预测，给 thesis：容器价值 = 应用层 LTV × vertical 数 × 治理溢价 |
