# Volvence 技术领先性与唯一性

> 我们做成了别人没做成的"关系运行时"（**Thin Prompt, Thick Runtime**）：让大模型从**每次都重新开始的陌生人**，变成**跨月跨年记得你、懂分寸、可被监管和接管的同一个 AI**。这一层已在 **closed-alpha 服务真实用户**，不是 demo，也不是论文设想。
>
> 它有完整的可信度链条：**对世界范围内 100 篇核心论文（尤其近 2–3 年）的研究结论 → 5 个收敛方向 → 我们 8 条核心技术推导 → 首席科学家世界级理论背书 → 一套可复核的实验机制 → 已经跑出来、经得起尽调的实验结果。** 本文按这条链展开，每一条结论都可被第三方独立复核。

---

## 一、这件事到底难在哪

长期关系型 AI 的难，不在"能不能聊天"，而在四件几乎没有现成方案的事：

- **记忆不是存档。** 难点不是把聊天记录塞回去，而是判断什么该长期记住、什么已过期、什么会污染当下决策。
- **判断不是规则。** 什么时候该关心、该推荐、该停下、该交给真人——靠 prompt 和 if/else 永远写不全，靠人工每次整理上下文又无法规模化。
- **成长不能压在嘴上。** 让 AI 越用越聪明，如果直接拿对话输出去做强化，会出现迎合、奖励投机、说一套做一套——这是行业公认的坑。
- **上线必须可治理。** 私域、母婴、医疗、金融场景下，AI 必须可审计、可删除、可随时人工接管，否则企业根本不敢采购。

**四件事任何一件单独都不容易；据我们对该领域的系统梳理，尚未见到把四者同时做到、还能跨垂直复用、还跑通验证的公开方案。这就是我们的位置。**

## 二、我们真正要解决的 7 个关系智能问题

长期关系型场景的本质不是"答得快"，而是持续回答下面 7 个问题。每一个，传统 prompt / RAG / workflow 都答不好，而我们都已转化为可测试的工程目标：


| #   | 真正的问题                          | 为什么传统方案答不好             | Volvence 如何应对（已验证方向）          |
| --- | ------------------------------ | ---------------------- | ----------------------------- |
| 1   | 用户**今天真正关心什么**，而不是只问了什么？       | 表层意图易抓，深层状态靠关键词猜不出来    | 跨会话的用户状态建模，区分"问出口的"与"真在意的"    |
| 2   | 用户兴趣、生活阶段、关系边界，**哪些应长期记住**？    | 全记会噪声爆炸，少记会丢边界         | 有取舍的长期记忆：可写入、可更新、可过期、可删除      |
| 3   | 这个品类**该补哪些知识、哪些缺口应优先学**？       | 人工喂料无法规模化，也不知道缺在哪      | 主动学习：少量真实互动下优先补最影响信任与转化的缺口    |
| 4   | 哪些 context **此刻重要、已过期、会干扰判断**？ | 人工每次整理上下文，塞少漏边界、塞多污染判断 | 上下文按当下关系状态动态取舍，而非一股脑回灌        |
| 5   | **什么时候在帮用户、什么时候在打扰用户**？        | 硬推转化伤关系，全程不推又没价值       | 关心 / 推荐 / 停止 / 接管的时机判断，可测、可复现 |
| 6   | 一次负面互动后，**如何修复信任**而不是继续转化？     | 单轮无状态，记不住"刚惹过用户"       | 信任破裂与修复、承诺、信任度可跨会话存活并影响后续     |
| 7   | 管理员**如何知道 AI 是否合格、越界、该重训**？    | 黑盒不可解释，出事无法追溯          | 决策可回溯、可审计、可接管、可回滚 + 上岗考核机制    |


**我们的差异不是把 context 拼得更好，而是把这 7 个问题做成一套会学习、会判断、可考试、可监管、可接管的运行时能力。**

## 三、研究源头：100 篇论文的结论与 5 个收敛方向

我们不是拍脑袋做 agent，而是先把这个领域读透：

- **100 篇 Cognitive AGI 核心论文 deep review**：10 个技术轴 × 10 篇，覆盖 2014–2026，每轴至少 6 篇 2024–2026 新作（论文清单与分轴笔记可在尽调时提供）。
- **49/100 篇可直接反哺工程 spec 与 benchmark 设计**——研究不是装点门面，而是直接进了我们的设计与测试（每篇均标注了对应的 spec / benchmark 入口）。
- **每个可应用方向都跑过完整消融 + A/B 验证，而非"读到就信"**：我们自建了消融测试与 A/B 实验框架和平台，对这 49 个前沿方向逐一拆解——去掉它系统是否真的变差、加上它增益从何而来，都要有数据支撑才进主干。
- **这套验证是持续的、双向的**：我们持续跟踪新发表的论文，从**正向**（新方法能否带来可测增益）与**反向**（现有能力在新基准/新反例下是否被证伪）两个维度，不断验证并加强我们的技术体系——研究地基不是一次性读完，而是随前沿滚动更新。
- **不止"研究论文"，主动学习支柱直接由该领域的世界级理论家支撑——杨柳博士**：其代表性成果横跨 **2011–2025、十余年持续深耕**，发表于 **JMLR、Annals of Statistics、NeurIPS、AISTATS、ALT** 等机器学习与数理统计**顶刊顶会**（含多篇第一 / 唯一作者），系统刻画了"用最少样本学到最多"的复杂度边界——这正是 Volvence 数据稀缺场景的理论命门（详见第五部分）。
- 上百篇研究最终**收敛到 5 个方向**，构成我们架构的理论地基：
  1. 把"预测误差"作为系统的一级原始信号，而非事后指标；
  2. 长期控制要发生在比文字更可控的内部层，而不是全压在自然语言（token）上；
  3. 记忆不是外挂数据库，而是架构本身的一部分；
  4. 有意义的行为结构应当从长期交互中**涌现**，而非预先硬编码规则；
  5. 自我修改必须有界，评估必须只读、不能反向变成学习源。

**这 5 个方向不是我们的发明，而是该领域前沿研究反复指向的共同结论——我们的工作是把它们落成可上线的工程。**

## 四、8 条核心技术推导和认知：从研究结论推到我们的架构

从上述研究出发，我们有一条完整、可被质询的推导链：

1. **下一代 AGI 首先是"认知型 AGI"**：不是更大的世界模型，也不是机器人本体，而是能形成目标、记忆、抽象、适应和恢复的认知系统。
2. **认知型 AGI 必须在线持续学习**：prompt / context / workflow 是临时脚手架，长期会输给能自己学习策略的系统。
3. **人的长期互动轨迹是下一代垂直数据护城河**：公共互联网数据见顶，机构数据有限，真实关系轨迹可再生、不可逆、长尾。
4. **持续学习需要强化，但不能直接压在自然语言输出上**：否则会带来奖励投机、推理污染、行为与解释分离。
5. **长期决策应发生在更可控的内部空间，而非全压在文字层**：系统要学的是"何时切换策略、复用哪类行动、如何跨时间尺度调整"。
6. **由此得到一套自洽闭环**：内部信号驱动学习、抽象来自长期交互、效率来自主动学习——优先选最有价值的样本与最关键的知识缺口。其中"主动学徒学习"这一前沿方向，杨柳博士正是开创者之一（下一节展开）。
7. **Thin Prompt, Thick Runtime**：prompt 只是表达层，真正的身份、记忆、边界、关系状态、审计、自修改门控都在运行时。
8. **这是一条独立于大模型公司的商业路径**：大厂卷通用 IQ，我们把人类关系轨迹变成可治理、可商业化、可长期复利的垂直数据层。

## 五、主动学习支柱的世界级理论支撑：杨柳博士

主动学习是 Volvence 架构的核心支柱之一（见第三部分收敛方向、第四部分推导 6、第七部分"主动学习效率"验证）。原因很直接：**真实关系互动数据稀缺且昂贵——每个用户只给出有限的真实对话，系统必须从极少样本里学到最多、并优先补最关键的知识缺口。这正是"主动学习"这一学科的核心命题：用最少的查询 / 标注达到给定精度。**

在这一支柱上，我们不只有论文综述，还有该领域的世界级理论家——杨柳博士。以下均为可公开核验的已发表 / 在审成果：


| 代表性贡献                                               | 发表（年份 / 等级）                          | 严格表述                                                                                        |
| --------------------------------------------------- | ------------------------------------ | ------------------------------------------------------------------------------------------- |
| **星数（star number）**                                 | JMLR（2015，CCF-A 顶刊）                  | 提出刻画低噪声下主动学习 minimax 标注复杂度的组合复杂度度量，并证明此前文献中几乎所有复杂度度量的最坏取值都等于星数                              |
| **Active Apprenticeship Learning + Eluder-star 维度** | ALT（2025）                            | 首创将主动学习与强化学习中的学徒学习相结合的新方向；提出 Eluder-star 维度刻画其样本复杂度；在 Massart / Tsybakov 噪声下给出鲁棒算法及样本复杂度上下界 |
| **可识别混合模型下的主动学习**                                   | Annals of Statistics（在审，数理统计顶刊，第一作者） | 推导满足正则条件的可识别混合模型下，主动学习算法风险的一般上界                                                             |
| **在线选择性采样的错误 / 查询权衡**                               | AISTATS（2021）                        | 建立在线选择性采样的一般理论框架，刻画"预测错误数"与"查询数"之间的最优权衡曲线                                                   |
| **批量主动学习（Buy-in-Bulk）**                             | NeurIPS（2013，CCF-A 顶会，第一作者）          | 给出按批查询标签的主动学习标注复杂度界，以及总查询数与轮数之间的权衡                                                          |
| **漂移分布下的主动学习**                                      | NeurIPS（2011，CCF-A 顶会，唯一作者）          | 在样本分布随时间变化的流式设定下，给出主动学习错误数 / 查询数上界与 minimax 下界                                              |


这些工作精确对应 Volvence 必须解决的真实问题：

- **样本极少**（用户只给有限真实互动）→ 标注 / 查询复杂度理论；
- **分布会漂移**（用户随时间变化）→ 漂移分布下的主动学习；
- **必须分轻重**（先补最关键的知识缺口）→ 选择性采样与最优查询权衡；
- **要与强化结合**（运行时的持续学习）→ Active Apprenticeship Learning。

> **严谨边界**：以上为杨柳博士在学术文献中可独立核验的研究成果（理论方向论文作者多按姓氏字母排序，上表已标注其为第一 / 唯一作者者）；Volvence 的工程实现是在这些理论指导下的独立工作，二者不等同、不混淆。但这意味着我们的"主动学习"支柱不是工程经验的拍脑袋，而是建立在该领域最前沿、可被同行验证的理论之上。

## 六、实验机制：怎么测才算数

我们没有用"看起来会聊天"当证据。我们的实验机制专门为**排除水分、经得起反问**而设计：

- **matched-control 对照设计**：每个能力都配对照组，排除"只是模型更会说话"的混淆变量——证明的是能力本身，不是口才。
- **多维 benchmark 套件**：覆盖长期记忆、内部决策、主动学习效率、多层记忆收敛、多人关系、用户心智建模、关系修复、长期信用归因 8 个维度，而不是一个笼统的"对话质量分"。**下一节的结果表即这套套件的逐项产出。**
- **盲评协议（blinded review）**：外部评审看不到任何 profile 标签，仅凭对话记录独立评分——避免"知道答案再打分"。
- **可回溯证据包（evidence bundle）**：每条结论都绑定 claim、随机种子、git sha、artifact、verdict 与置信区间，可逐条、可复现地核查。
- **契约测试 + 无回归**：架构每次升级都跑契约测试，保证旧能力不被新改动破坏。

**一句话：我们交付的不是分数，而是一套别人可以拿过去自己复跑的验证方法。**

## 八、为什么是"唯一"

我们的"唯一"来自结构性位置：

- **基础大模型公司不会做**：他们卷通用 IQ，不会为每个垂直关系单独建立长期记忆与治理。建立关系，有的时候需要"拒绝用户"、需要设立边界——这些都和大公司的风控、业务方向正交。
- **垂直 App 做不到**：自训模型成本过高；只拼 prompt / RAG / workflow，又支撑不了长期关系与可审计。
- **学术界停在论文**：相关思想散落在上百篇论文里，但没人把它收敛成可上线、可测试、可商业化的运行时——而我们做到了（第三到第七部分即证据）。
- **我们正好站在这唯一的位置**：把"记住人、懂分寸、能成长、可监管"做成一层可跨垂直复用的中间层——大模型公司不会做、产品公司做不了，但每个长期关系型产品都必须有的一层。

这一层一旦建成，就是**可复利的护城河**：同一套中间层承载多个垂直产品，真实关系轨迹是可再生、不可逆、长尾的数据资产，越用越深，越深越难被替代。这就是 **Thin Prompt, Thick Runtime** 的商业含义——价值不在每一句 prompt，而在那层越长越厚的运行时。
